# AI Agent - 第 25 课：记忆工程深水区：写入判定、冲突、遗忘、用户画像与治理

## 学习目标

- 理解长期记忆系统为什么常常是 Agent 后期最脏、最难管的部分。
- 搞清楚记忆系统的关键问题不是“能不能存”，而是“该不该存、怎么更新、怎么失效”。
- 掌握写入判定、冲突解决、遗忘机制、用户画像治理这些核心问题。
- 能设计一个不会越用越脏的记忆系统。

## 内容讲解

### 1. Memory 系统最容易被浪漫化

很多人一谈记忆，会想象成：

- Agent 越用越懂你
- 用得越久越聪明
- 会逐渐形成“长期关系”

这些想象不是完全错，但工程上更真实的画面通常是：

- 记进去了太多没用信息
- 旧偏好和新偏好冲突
- 一次错误回答被当成长期事实
- 用户画像逐渐污染

所以 Memory 的核心挑战，从来不是存储，而是治理。

### 2. 长期记忆系统最常见的坏掉方式

1. 什么都记
2. 记住了错误事实
3. 旧记忆不失效
4. 用户偏好和任务事实混在一起
5. 检索时总召回过时内容
6. 敏感信息误入长期存储

### 3. 写入判定为什么是第一关键点

如果没有写入判定，系统通常会退化成：

- 会话日志仓库
- 噪声垃圾堆

所以不是“每轮都写”，而是要做：

- 候选提取
- 价值判断
- 结构化归一
- 冲突检测
- 最终写入

### 4. 哪些信息通常值得记

比较适合长期沉淀的包括：

- 稳定偏好
- 长期约束
- 重复出现的实体关系
- 高价值经验总结
- 跨会话任务进展

### 5. 哪些信息通常不应该记

- 一次性情绪表达
- 临时故障状态
- 未验证推断
- 冗长原始对话
- 高敏感原文数据

### 6. 记忆冲突为什么会越来越频繁

随着使用时间增长，你几乎必然会遇到：

- 用户以前喜欢中文，现在要求英文
- 老项目结构变了
- 旧 SOP 被新流程替代
- 上次总结和本次事实不一致

如果记忆系统没有冲突解决机制，就会频繁召回错误信息。

### 7. 冲突解决的常见策略

可以从四个维度判断：

- 时间新旧
- 来源可信度
- 明确覆盖关系
- 是否需要人工确认

并不是所有冲突都该自动覆盖。

例如“用户偏好”通常可以新覆盖旧。  
但“组织制度”或“业务规则”更适合保留版本和来源。

### 8. 遗忘机制为什么是必须的

记忆系统如果只会增长，不会遗忘，最终一定出问题。

遗忘常见策略包括：

- TTL
- 置信度衰减
- 最近未使用淘汰
- 人工标记失效
- 版本替换

### 9. 用户画像为什么危险

用户画像看起来很有价值，但也最容易变成风险源。

常见风险：

- 用户画像过度推断
- 私密信息长期保存
- 模型把“猜测”当成事实
- 业务策略与真实用户意图不一致

所以用户画像应尽量：

- 结构化
- 可解释
- 可删除
- 可覆盖
- 可审计

### 10. 记忆系统和 RAG 最大的区别

RAG 面向外部知识。  
Memory 面向内部持续关系和状态。

如果把这两者混在一起，系统就会出现：

- 外部知识和用户偏好相互污染
- 召回时难以做优先级判断
- 一旦错误写入，影响范围过大

### 11. 记忆系统最稳的设计原则

1. 分层存储
2. 只沉淀高价值信息
3. 记忆条目带来源
4. 记忆条目带时间
5. 支持覆盖和失效
6. 不保存不必要的敏感原文

## 小结

长期记忆不是“越多越好”。  
真正成熟的记忆系统，更像一个带治理能力的知识面板。
