# AI Agent - 第 28 课：成本工程：模型路由、缓存、批处理、上下文预算与评估门禁

## 学习目标

- 理解 Agent 成本为什么常常不是线性增长，而是乘法增长。
- 掌握常见成本控制手段：模型路由、缓存、批处理、上下文预算、评估门禁。
- 学会区分“高质量必要成本”和“系统设计造成的浪费”。

## 内容讲解

### 1. Agent 成本为什么容易失控

因为它不是一次模型调用，而往往是：

- 多轮模型调用
- 多次工具调用
- 多段检索
- 多份上下文拼装

所以一个用户请求，最后消耗的并不是一次 token，而是一整条推理轨迹。

### 2. 最常见的隐性成本源

1. 上下文过长
2. 重复检索
3. 多 Agent 重复思考
4. 大模型被滥用于低风险步骤
5. 无意义重试
6. 评估体系太重

### 3. 模型路由为什么是第一省钱点

不是所有步骤都需要最强模型。

可以按任务分层：

- 分类 / 路由 -> 小模型
- 摘要 / 压缩 -> 中模型
- 复杂推理 / 高风险决策 -> 大模型

### 4. 缓存为什么不只是 RAG 结果缓存

在 Agent 里常见可以缓存的内容包括：

- 检索结果
- 工具结果
- 中间摘要
- 结构化计划
- 评估结果

### 5. 上下文预算是最容易被忽略的成本控制手段

很多团队会讨论模型单价，却忽略：

真正导致 token 爆炸的往往是上下文工程失控。

控制上下文预算，通常比单纯换模型更直接。

### 6. 评估门禁为什么能防止“越改越贵”

一个成熟团队不应该只在效果下降时回滚，还要在：

- 成本明显上升
- 平均步数明显增加
- 工具调用数暴涨

这些情况下触发门禁。

## 小结

Agent 成本治理的核心不是抠单次 token，而是控制整条执行轨迹的复杂度。
