# AI Agent - 第 30 课：AI Agent 线上排障手册：幻觉、循环调用、空转、卡死、观测与回放

## 学习目标

- 建立一套线上排障心智模型，知道 Agent 出问题时该先看什么、后看什么。
- 学会区分模型问题、工具问题、检索问题、状态问题和编排问题。
- 掌握几类典型故障：幻觉、循环调用、空转、卡死、错误恢复失败。

## 内容讲解

### 1. 线上排障最怕什么

最怕看到一句抽象反馈：

- “Agent 不太稳定”

这类问题如果没有 trace，几乎没法定位。

### 2. 一条最实用的排障原则

先不要问：

- 模型为什么这么笨

先问：

- 它到底看到了什么
- 它做了什么
- 它为什么这么做
- 它在哪一步开始偏

### 3. 几类最常见故障

1. **幻觉**
   - 没证据却说得像真
2. **循环调用**
   - 工具不断重试或相互调用
3. **空转**
   - 一直在整理、总结、搜索，但不收敛
4. **卡死**
   - 等待外部结果或状态丢失
5. **错误恢复失败**
   - 任务恢复后走错分支或重复副作用

### 4. 排障顺序建议

1. 看 trace
2. 看上下文装配
3. 看工具选择
4. 看参数与结果
5. 看检索证据
6. 看状态机与检查点
7. 看超时、重试和队列

### 5. 幻觉很多时候不是模型“凭空编”

真实生产里大量“幻觉”其实是：

- 检索证据错
- 旧记忆被召回
- 工具结果不完整
- 上下文被截断

### 6. 循环调用最常见的根因

- 没有终止条件
- 工具错误语义太差
- 重试和正常执行没区分
- 模型总觉得“再试一次可能行”

### 7. 空转为什么常被误判

空转的外观通常很像“系统很努力”：

- 一直在搜索
- 一直在总结
- 一直在改写 query

但真正问题是：

- 没有收敛判定
- 没有信息增益判断

### 8. 卡死为什么往往和等待状态有关

任务卡死不一定是线程卡死。  
更多时候是任务状态停在：

- waiting_callback
- waiting_human
- waiting_tool_result

但系统没有 watcher 去推进。

### 9. 回放为什么重要

没有回放，你只能靠日志猜。  
有了回放，你才能真正重建：

- 输入
- 上下文
- 工具链
- 中间决策
- 最终结果

## 小结

线上排障最重要的不是“经验丰富”，而是：

- 你的系统是否留下了足够证据
- 你的状态流转是否可追踪
- 你的工具和上下文是否可回放

4. 空转和努力工作看起来为什么很像？
5. 为什么回放比单纯错误日志更关键？
