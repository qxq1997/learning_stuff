# Claude Code 源码学习 · 章节目录与导读

## 写在前面

Claude Code 是 Anthropic 官方的 CLI 形态 Agent。它的 npm 包 `@anthropic-ai/claude-code` 里其实就有一份打包后的 `cli.js`(虽然经过 minify 与混淆),社区也已经做过大量逆向分析。这门课不是要你**抄一份 Claude Code**,而是借助这份"几乎公开"的实现去回答一个更本质的问题:

> 一个真正能落地的 Agent 系统,在工程上到底长什么样?

我们不会逐行读 cli.js——那是 minify 过的 4 万行 JS,直接读没有任何意义。我们要做的是:

1. **从外向内**:先看它**对外暴露了什么**(命令行参数、子命令、配置)。
2. **从行为推机制**:观察它**怎么响应输入**,然后猜它内部应该有什么。
3. **从公开资料校准**:对照 docs.claude.com、官方 release notes、社区逆向文章。
4. **从原理上抽象**:剥离实现细节,提取**可复用的 Agent 设计模式**。

最后的产物,是一份你能在白板上画出来、能口头讲清楚、还能自己复刻一个 mini 版的 Agent 设计图。

## 阶段一 · 基础认知(2 课)

打掉"它就是个套壳 ChatGPT"的错觉。

- **01 · Claude Code 的项目定位与"Agent"的本质**
  搞清楚 Agent 和 Chat 的边界:为什么"能调用工具的对话"才叫 Agent,Claude Code 解决了 Claude.ai 没法解决的什么问题。

- **02 · 整体架构鸟瞰:进程、模块、数据流**
  画一张全景图。从你按下回车那一刻,直到屏幕上吐出第一个字符,中间到底经过了哪些环节。

## 阶段二 · Agent 循环核心(3 课)

理解 Agent 之所以是 Agent 的那个 `while` 循环。

- **03 · Agent 主循环**:对话不是请求-响应,是循环。
- **04 · Tool Use 循环的递归本质**:`tool_use` 与 `tool_result` 互相喂养,直到模型说"我说完了"。
- **05 · 流式响应处理**:SSE、event 解析、增量渲染、被打断怎么办。

## 阶段三 · Prompt 工程(2 课)

System Prompt 不是写出来的,是**拼出来的**。

- **06 · System Prompt 的构造**:静态部分 + 动态注入 + CLAUDE.md 多层合并。
- **07 · 上下文管理与压缩**:auto-compact、缓存命中、消息裁剪策略。

## 阶段四 · 工具系统(3 课)

工具是 Agent 的手脚。这一层决定了 Agent 能干多少事。

- **08 · 工具的定义与注册**:为什么"工具"本质上就是一个 JSON Schema + 一段执行代码。
- **09 · 工具调度与执行**:并发、超时、错误传播、权限拦截。
- **10 · 文件类工具的设计取舍**:为什么 Claude Code 选择 Read/Edit/Write 三件套,而不是一个万能 Patch。

## 阶段五 · 扩展机制(4 课)

第三方怎么进来。

- **11 · Subagent 与 Task 工具**:子 Agent 不是为了"更聪明",是为了**保护主上下文**。
- **12 · Hook 系统**:用户脚本怎么挂到 PreToolUse / PostToolUse / Stop 等生命周期点。
- **13 · Slash Commands 与 Skills**:`/foo` 是怎么从一行输入变成一次工具调用或一段提示注入的。
- **14 · MCP 集成**:Model Context Protocol 这一层抽象,究竟解决了什么问题。

## 阶段六 · 权限与安全(2 课)

Agent 越强,越要"管得住"。

- **15 · Permission 模型**:allow / ask / deny 三态决策,以及 plan / accept-edits 等模式。
- **16 · 提示词注入防御**:为什么"工具结果里的指令"必须被当作不可信数据。

## 阶段七 · 工程实现(3 课)

让它跑得稳的工程细节。

- **17 · 会话持久化**:JSONL 形式的 transcript、resume 的恢复策略。
- **18 · TUI 渲染与体验**:Ink + React、流式 Markdown、Ctrl-R / Esc-Esc 这些细节是怎么实现的。
- **19 · Memory / CLAUDE.md 机制**:三层 memory(用户、项目、本地)怎么合并、覆盖、注入。

## 阶段八 · 综合(1 课)

- **20 · 从零设计一个 Agent CLI**:把前面 19 课的所有抽象,搓成一份你自己的设计文档。

## 学习方法约定

1. **每课必有一个核心问题**。读完之前先把问题盖住,看自己能不能用一句话答上来。
2. **画图比念字重要**。每个机制至少配一张 ASCII 流程图或时序图。
3. **对照 EMF**。你正在做的 EMF 流程编排平台和 Claude Code 在"DAG/循环 + 工具调用 + AI 赋能"这件事上有大量可类比之处,我会在合适的地方主动做对照。
4. **绝不跳课**。如果某一课的问题答不上来,我们生成 `XXb_补充.md` 而不是硬推。
