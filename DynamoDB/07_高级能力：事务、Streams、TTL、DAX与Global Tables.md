# DynamoDB - 第 7 课：高级能力：事务、Streams、TTL、DAX 与 Global Tables

## 学习目标（本节结束后你能做到什么）

- 知道 DynamoDB 的事务、Streams、TTL、DAX、Global Tables 各自在解决什么问题。
- 明白这些能力虽然很强，但并不意味着都该放在核心主链路里。
- 能把这些高级能力和真实工程需求联系起来，而不是停留在名词层面。

## 内容讲解（核心概念，用类比、例子、图示说清楚）

### 1. 为什么这些能力放到最后学

因为它们都属于：

- 很重要
- 很常被问
- 但如果前面主键、分区、索引、容量都没理解好，学起来很容易空转

真正稳的顺序应该是：

先会建模，再会扩展，再来理解这些增强能力。

### 2. 事务：DynamoDB 不是不能做事务，而是不该把事务当常态思路

DynamoDB 支持事务读写，这一点很多人知道，但经常被误用。

事务的价值在于：

- 多条 item 写入要么都成功，要么都失败

这对某些跨 item 一致性场景非常重要。

但事务不是 DynamoDB 最鼓励的主路径，原因通常有：

- 成本更高
- 延迟更高
- 心智负担更重

所以 DynamoDB 的工程思路通常是：

- 能用单 item + 条件写解决，优先别上事务
- 真有必要跨 item 原子提交，再用事务

### 3. Streams：不是消息队列，但很像变更日志流

Streams 的核心价值是：

**把表中的变更，变成一条可消费的事件流。**

典型用途：

- 变更同步到搜索索引
- 写审计日志
- 做异步下游投递
- 触发 Lambda

它不是传统 MQ 的完全替代品，但非常适合做：

“数据库变更后的异步下游动作”

这和 Redis Streams 的使用场景有点像，但语义更贴近“数据库 CDC 风格的变更流”。

### 4. TTL：让数据自己过期，但不要把它理解成精确定时器

TTL 很适合处理：

- 会话
- 临时状态
- 过期 token
- 短期业务缓存

但要特别注意：

TTL 更像“过期后系统会异步帮你回收”，而不是：

“到点立刻精确删除”

所以如果你的业务要求：

- 到 10:00:00 必须马上失效

那 TTL 不能被当成唯一保障，更稳的做法是：

- 业务侧也做过期判断

### 5. DAX：它不是 DynamoDB 本体，而是前面的缓存层

DAX 可以理解成 DynamoDB 官方体系里的一个加速层。

它主要解决的是：

- 高频读
- 对延迟特别敏感
- 希望少改应用逻辑

你可以把它理解成：

**一个更贴着 DynamoDB 的读缓存。**

但它也不是免费午餐：

- 增加系统组件
- 增加一致性理解成本
- 不适合所有场景

如果你只是先建立整体认知，到这里知道它的定位就够了；如果你想更扎实地理解：

- 它到底缓存的是 item 还是 query 结果
- 它为什么主要优化 eventually consistent read
- 它和 Redis、本地缓存、强一致读之间到底是什么边界

可以继续看后面的第 10 课，那一节会把 DAX 单独拆开讲。

如果本来业务读写模式就不适合缓存，DAX 也不是神药。

### 6. Global Tables：多 Region 复制能力

如果你的系统要跨地域部署，Global Tables 很有吸引力。

它主要解决：

- 多 Region 写入
- 跨区域就近访问
- 区域级故障时的数据可用性

但你也要意识到代价：

- 冲突处理复杂度上升
- 一致性语义要重新理解
- 成本会更高

所以它更像是一种：

**为跨 Region 业务提供能力的高级武器**

而不是默认打开的选项。

### 7. 为什么这些高级能力最容易被“看起来很强”误导

因为它们名字都很诱人：

- 事务
- 流
- 全局表
- 加速缓存

但工程上最关键的问题始终是：

**这个能力，是在帮你解决核心矛盾，还是在增加系统复杂度？**

例如：

- 如果只是一个单 item 状态机，事务可能完全没必要
- 如果只是简单过期控制，TTL 很好；但如果要精确调度任务，TTL 不是定时任务系统
- 如果跨地域读并不多，Global Tables 未必值回复杂度

### 8. 学这些特性的正确姿势

不是记“它有什么功能”，而是记：

- 它在替代什么
- 它引入了什么新成本
- 它适合放在主链路还是边缘链路

这才是工程判断。

## 小结（3-5 条关键点）

- DynamoDB 支持事务，但事务不是最优先的默认解法，单 item + 条件写往往更自然。
- Streams 更像变更日志流，特别适合做异步下游处理。
- TTL 适合做自动过期回收，但不能当精确调度器。
- DAX 是前置缓存层，适合高频低延迟读场景，不是万能性能补丁。
- Global Tables 提供跨 Region 能力，但也带来一致性、冲突和成本上的复杂度。
