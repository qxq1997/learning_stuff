# 11 系统学习路线

## 1. 第一遍: 先建立概念主线

第一遍不要追文件名,先建立这条概念链:

```text
Message
  -> DAG
  -> ready 判断
  -> 并发调度
  -> command 执行
  -> 新 Message
```

这一遍要回答:

1. 为什么 `Message` 可以代表任务。
2. 为什么一批 `Message` 可以推导出 DAG。
3. 为什么 command 和调度要分层。
4. 为什么动态 workflow 最终还是生成新 `Message`。

这一遍最重要的是不要被 Databricks、Storage、Catalogue 这些具体执行能力带偏。它们属于执行平面;EMF 的核心在控制平面:

```text
Message / DAG / ready / state / priority / retry / timeout / batch-run lifecycle
```

先把控制平面讲清楚,再看具体 command 怎么调用外部系统。

## 2. 第二遍: 学消息模型

重点理解:

```text
Message
batch_id
run_uuid
order_id
parents
command
parameters
```

尤其要记住:

```text
DAG 依赖靠 parents 引用父任务的 order_id,
不是靠 msg_id.
```

这一遍可以把 `Message` 当成一份“小型任务协议”来学:

```text
我要执行什么: command
我属于谁: batch_id / run_uuid
我在图里的位置: order_id
我依赖谁: parents
我怎么执行: parameters
```

## 3. 第三遍: 学 DAG 调度

重点不是“调度器怎么写代码”,而是这几个机制:

- `parents` 是 join barrier。
- `run_date` 可以延迟执行。
- 状态机决定节点是否还能被调度。
- `batch_priority` 决定多个 ready 节点谁先跑。
- `max_messages`、slot、semaphore 决定并发边界。

读的时候可以问五个问题:

1. 哪些节点 ready?
2. 为什么这个节点还不能跑?
3. 多个 ready 节点怎么排序?
4. 并发会不会破坏依赖?
5. 失败后怎么回到调度系统?

## 4. 第四遍: 学 OE-RUN

`OE-RUN` 是理解 workflow 如何从配置或表变成 DAG 的核心。

目标是理解:

```text
process_tasks / catalogue resolution
  -> Run
  -> workflow messages
  -> enqueue
  -> runtime DAG
```

这一遍要重点想:

- 顶层 workflow 和子 workflow 的区别是什么。
- 子 workflow 如何接回父 workflow。
- completion token 为什么需要存在。
- label filtering 为什么会影响 DAG 结构。
- 子 workflow 为什么需要 `order_id` 命名空间。
- 子 workflow 沿用同一个 `run_uuid` 和新开 `run_uuid` 的区别。

## 5. 第五遍: 学命令和模块方式

命令层的学习重点:

```text
command 是协议名
module 是实现
parameters 是输入协议
service 是业务依赖
```

读命令时重点看:

```text
这个命令解决什么业务问题?
它需要哪些参数?
它有没有副作用?
它会不会生成新 Message?
它失败后是否可以 retry?
```

## 6. 第六遍: 学运行治理

主线通了之后,再看这些治理能力:

- retry。
- timeout。
- cancel。
- batch cleanup。
- logging。
- audit log。
- priority scheduling。
- enabled / disabled label filtering。
- process task order validation。
- 动态 fan-out / fan-in。
- completion token。

这些能力不是独立系统,而是围绕 message 状态机和 batch/run lifecycle 展开。

## 7. 第七遍: 最后学外部系统集成

最后再深入:

- Databricks。
- Storage。
- Service Bus。
- Catalogue。
- `LoadInfo`。
- UDF。

这些是业务基础设施集成层。它们很重要,但如果过早进入,容易把 EMF 误解成“Azure 工具集合”,而看不到它真正的 workflow engine 主线。

## 推荐复习方式

面试前按这三个层次复习:

```text
一句话: Message 驱动的动态 DAG 工作流引擎
三分钟: 入口、建图、调度、执行、治理、动态扩展
深入问答: retry、子 workflow、label filtering、并发、命令插件化
```
