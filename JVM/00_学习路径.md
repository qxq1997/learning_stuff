# JVM 学习路径与知识图谱

## 学习定位

很多后端工程师学 JVM，容易走向两个极端：

- 只会背八股：知道双亲委派、Minor GC、Full GC、G1、ZGC 这些名词，但说不清它们为什么存在、解决什么问题。
- 只看局部源码：一上来就钻 HotSpot 细节，结果整体地图没有建立起来，知识点彼此孤立，越学越碎。

这条学习路径的目标，不是把你变成“会背 JVM 术语的人”，而是帮你建立一张完整的运行时地图，让你真正理解：

- Java 程序是怎么从源码变成可执行程序的
- JVM 在运行时到底管了哪些事
- 对象为什么会被回收，为什么有那么多垃圾回收器
- G1 和 ZGC 到底各自在解决什么问题
- 为什么有些 GC 追求吞吐，有些 GC 追求低停顿
- 面试里 JVM 到底应该怎么讲，线上排障时又该看什么

学完这一套，你应该能做到：

- 把 JVM 放回整个 Java 运行时链路里理解，而不是只盯着 GC
- 说清类加载、内存区域、对象创建、可达性分析、分代回收之间的关系
- 真正看懂 G1 和 ZGC 的设计动机，而不是只会背执行步骤
- 对“为什么这样设计”形成直觉，而不是只记“怎么定义”

## 知识图谱（从零到专家）

1. 先建立整体认知
   - JVM 是什么
   - JDK、JRE、JVM 分别是什么
   - Java 程序从源码到运行经历了什么
2. 再理解运行时内存模型
   - 程序计数器
   - 虚拟机栈
   - 本地方法栈
   - 堆
   - 方法区
   - 对象创建、分配、访问
3. 再理解类加载机制
   - 类加载生命周期
   - 双亲委派
   - 不同类加载器的职责
   - 类初始化和字节码执行的关系
4. 再进入垃圾回收基础
   - 为什么需要 GC
   - 引用计数为什么不够
   - 可达性分析
   - GC Roots
   - 分代思想
   - 三色标记
5. 再看垃圾回收器全景
   - Serial / Parallel / CMS / G1 / ZGC 各自解决什么问题
   - 吞吐量、停顿时间、内存占用分别代表什么
   - 现代 GC 共同阶段：初始标记、并发标记、重新标记、清理与转移
6. 再补上 CMS 这条经典并发回收路线
   - `ParNew + CMS`
   - 初始标记、并发标记、Remark、并发清理
   - 浮动垃圾、碎片、退化
7. 再深入 G1
   - Region
   - RSet / Card Table
   - Young GC
   - 全局并发标记
   - Mixed GC
8. 再深入 ZGC
   - Page
   - 染色指针
   - 读屏障
   - 重定位与自愈
   - 为什么它能把停顿压得很低
9. 最后进入工程实践
   - 如何选择 GC
   - 用具体数字判断 GC 是否真的有问题
   - 常见 GC 日志怎么看
   - 什么情况下调 G1，什么情况下考虑 ZGC
   - 面试答题框架
10. 再补上线上排障工具地图
   - `jps` / `jstat` / `jstack` / `jmap` / `jcmd`
   - `JConsole` / `VisualVM`
   - `MAT` / `Arthas` / `GCeasy`
11. 再理解常用 JVM 参数
   - `-Xms` / `-Xmx`
   - `-Xmn` / `-XX:NewRatio`
   - `-XX:SurvivorRatio`
   - `-XX:+HeapDumpOnOutOfMemoryError`
   - `-XX:HeapDumpPath`
12. 最后做线上故障实战
   - 怎么定位 Java 进程
   - 怎么查死锁
   - Heap Dump 怎么生成
   - OOM 怎么结合 MAT 分析
13. 扩展专题：历史 CMS 系统排障
   - 评价指标与 GC Cause
   - 空间震荡、显式 GC、MetaSpace OOM
   - 过早晋升、CMS Old GC 频繁、单次停顿过长
   - 收集器退化、堆外内存 OOM、JNI 问题
14. 扩展专题：G1 生产排障
   - 评价指标与 G1 关键 GC Cause
   - Young GC 频繁、Humongous Allocation 与 Young 区压力
   - Mixed GC 频繁、Remark 过长与 RSet 压力
   - To-space exhausted、Evacuation Failure 与 Full GC
15. 扩展专题：线上 JVM 调优案例复盘
   - 接口 GAP 大但业务代码本身不慢
   - 参数误配导致的特殊 OOM
   - RES 超过 `-Xmx` 的 Native Memory 问题
   - 上线后 YGC 暴涨
   - JVM 粗调的方法和边界

## 为什么这样排序

JVM 这门东西最容易学乱的原因是：大家一上来就学 GC，而且一学就是 G1、CMS、ZGC 的各种名词。

问题在于，如果你没有先理解：

- JVM 的整体职责是什么
- 对象在 JVM 里是怎么创建和存活的
- 为什么对象最终需要被回收
- 什么叫可达性分析，什么叫分代

那么后面的 G1 和 ZGC 就会变成一堆很难消化的术语：

- Region、RSet、SATB
- 染色指针、读屏障、自愈

这些东西单独背当然也能背，但你很难真正明白它们“为什么存在”。

所以正确顺序一定是：

1. 先建立 JVM 整体地图
2. 再理解对象与内存
3. 再理解 GC 基础算法
4. 先抽象现代 GC 的共同阶段
5. 先补上 CMS 这条经典并发回收路线
6. 再进入 G1 和 ZGC 这些现代回收器设计
7. 最后再进入参数、工具和线上故障实战

这样你在学 G1 和 ZGC 时，就不是“硬吃黑话”，而是在回答一个个自然问题：

- 为什么 G1 要用 Region
- 为什么 G1 要引入 RSet
- 为什么 ZGC 要用染色指针
- 为什么 ZGC 能把 STW 压得这么低

## 学习方法建议

- 每学一个 JVM 概念，都先问一句：它到底在解决什么问题。
- 学到具体机制时，不要先背流程，要先问：如果没有这个机制，会慢在哪里、错在哪里、卡在哪里。
- 学 GC 时，一定把“对象生命周期”“引用关系”“标记”“复制/清理/压缩”串成一条链，而不是把名词拆着背。
- 学 G1 和 ZGC 时，要先抓设计目标，再抓关键结构，再抓执行流程，最后才记参数和细节。
- 如果是面试导向，永远优先练“用自己的话讲清楚”，而不是直接背教材原句。

## 学完后你应该能回答的问题

- JVM 到底是什么，它和 JDK、JRE 的关系是什么。
- 为什么 Java 需要垃圾回收，为什么引用计数法不够。
- G1 和 ZGC 到底分别想解决什么问题。
- 为什么 G1 追求可预测停顿，ZGC 追求极低停顿。
- RSet、SATB、染色指针、读屏障分别在解决什么问题。
- 什么情况下继续用 G1，什么情况下值得考虑 ZGC。
- 服务器上怎么快速定位 Java 进程、怎么看 JVM 参数和线程快照。
- 出现 OOM 后为什么 Heap Dump 是最关键的现场，以及怎么用 MAT 把根因挖出来。

## 推进规则

- 如果你从零开始，建议严格按顺序学习。
- 如果你已经会基础八股，重点补第 4、5、6、7、8、9 课。
- 如果你的目标是面试和生产排障，`08` 到 `11` 以及 `08b` 这几课最好连续看完，这样才能把“GC 原理”和“线上落地排查”真正串起来。
- 如果你维护的是历史 Java 8 / CMS 系统，建议先看 `05c` 建立 CMS 总体认知，再系统学习 `12` 到 `15` 这四课，它们会把“CMS 为什么出问题”和“怎么定位根因”真正讲透。
- 如果你现在主要在线上维护的是 G1 服务，建议先看 `06` 建立原理认知，再把 `16` 到 `19` 这四课连着看完，这样会把“G1 为什么会抖、为什么会 Humongous、为什么会 Evacuation Failure”真正串起来。
- 如果你想把“JVM 参数、GC 指标、Heap Dump、线上事故”真正串起来，`20` 这课适合在 `11` 和 `16~19` 之后看，它会把零散案例沉淀成更稳定的排障套路。
