## 内容讲解（核心概念，用类比、例子、图示说清楚）

### 1. 为什么 Java 需要垃圾回收

在 C/C++ 世界里，程序员通常要自己管理内存：

- 什么时候申请
- 什么时候释放

这样做灵活，但也非常容易出问题：

- 忘记释放，造成内存泄漏
- 提前释放，造成野指针
- 重复释放，造成崩溃

Java 想把程序员从这类低层内存管理里解放出来，所以 JVM 引入了垃圾回收机制。

垃圾回收本质上在做一件事：

**自动找出以后不再需要的对象，并回收它们占用的内存。**

### 2. 什么叫“垃圾”

这里的“垃圾”不是说对象内容没用，而是说：

**这个对象以后再也不可能被程序访问到了。**

只要还有一条有效引用链能从程序运行的根部走到它，它就不算垃圾。  
只有彻底不可达，才可以被回收。

所以 GC 的核心从来不是“看对象值”，而是“看对象还能不能被访问到”。

### 3. 为什么引用计数法不够

引用计数法很直观：

- 每个对象记录有多少个引用指向它
- 计数为 0 就回收

但它有一个经典问题：循环引用。

例如：

- A 引用 B
- B 引用 A
- 外部已经没有任何引用指向它们

这两个对象明明已经没用了，但计数都不是 0，于是永远回收不了。

所以 JVM 主流并不依赖引用计数法，而是使用可达性分析。

### 4. 可达性分析到底是什么

可达性分析的思想可以理解成：

- 先找一组“肯定活着的起点”
- 从这些起点出发，沿着引用关系往下走
- 能走到的对象，都算还活着
- 走不到的对象，才可能被回收

这些起点就叫 GC Roots。

常见的 GC Roots 包括：

- 虚拟机栈里正在使用的引用
- 本地方法栈中的引用
- 类静态属性引用的对象
- 常量引用的对象等

### 5. 分代思想为什么成立

JVM 的垃圾回收之所以常常分新生代、老年代，不是因为对象“年纪大就必须尊重”，而是因为实践里发现：

**大多数对象都是朝生夕死的，少数对象会活很久。**

例如：

- 一次请求里临时创建的对象，往往很快就死了
- 应用缓存、单例、长生命周期集合对象，会活得很久

于是 JVM 形成了一个非常实用的思路：

- 新生代：专门处理大量短命对象
- 老年代：放长期存活对象

这样不同代可以用不同策略，提高整体效率。

### 6. 标记、清除、复制、整理分别是什么意思

这些是 GC 世界的基本动作。

#### 标记

先找出哪些对象活着。

#### 清除

把没标记到的对象空间释放掉。

#### 复制

把存活对象复制到另一块区域，原区域整体清空。

#### 整理

把存活对象往一侧搬，消除内存碎片。

不同垃圾回收器，本质上就是在不同区域、不同阶段，对这些动作做不同组合。

### 7. 三色标记算法是什么

三色标记是现代并发标记里特别重要的思想。

你可以把对象理解成三种颜色：

- 白色：还没被检查
- 灰色：这个对象确定是活的，但它引用出去的对象还没全部检查完
- 黑色：这个对象以及它引用出去的对象都检查完了

整个过程就是：

1. 从 GC Roots 开始，把起点放成灰色
2. 取一个灰色对象，检查它的子引用
3. 子引用对应的对象如果还没处理，就变灰
4. 当前对象检查完后变黑
5. 最终还白着的对象就是不可达对象

三色标记本身不难，难点在于：

**标记过程中，用户线程还可能在改引用。**

### 8. 为什么并发标记会漏标

假设 GC 线程正在标记，对象关系却被业务线程改了。

典型问题是：

- 某个白色对象原来由灰色对象指着
- 后来这条引用断了
- 同时又被黑色对象新引用了

这时它可能既没被灰色阶段扫到，也不会被黑色对象重新扫到，于是被漏标。

所以现代 GC 在并发标记时，一定要引入额外机制来修补引用变化带来的问题。

后面学 G1 时，你会看到 SATB；学 CMS 时会看到增量更新；学 ZGC 时会看到读屏障等更激进的并发设计。

### 9. 为什么这节课对 G1 和 ZGC 尤其重要

如果你没理解：

- 可达性分析
- GC Roots
- 分代思想
- 三色标记
- 漏标问题

那 G1 和 ZGC 后面的术语会非常难懂。

比如：

- G1 为什么需要 SATB
- ZGC 为什么需要读屏障

这些都不是凭空冒出来的，它们都是在解决“并发标记时引用会变化”这个根问题。

## 小结

- 垃圾回收要解决的是自动识别并回收以后再也访问不到的对象。
- JVM 主流不用引用计数法，是因为它难以处理循环引用等问题。
- 可达性分析的核心是：从 GC Roots 出发，能走到的对象算存活。
- 分代回收的经验基础是：大多数对象短命，少数对象长寿。
- 三色标记是现代并发标记的重要思想，而引用变化带来的漏标问题，是 G1 和 ZGC 等回收器设计的关键背景。
