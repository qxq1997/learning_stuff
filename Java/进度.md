## 当前学习假设

- 你有后端开发经验，但默认不假设你已经系统梳理过 JVM 和 JUC。
- 学习目标不只是面试八股，更包括真实项目里的线程池、异步任务、锁竞争、上下文透传与线上排障。
- 语言层的函数式风格也要补齐，因为它直接影响 `Stream`、异步编排和代码表达能力。
- 集合不只记 API：需要从数据布局、键契约、迭代视图与并发语义理解选型代价。
- 基础概念要能落到运行链路：字节码/类加载、类型转换、对象语义、异常资源边界与 I/O 模型都要能解释工程后果。
- 教学会优先贴近后端工程问题，例如请求线程、线程池复用、上下文传递、异步调用、锁和同步器。
- JUC 素材中的 `ThreadLocal` 与 AQS 主干已有专章覆盖，本轮只增补缺失的 JMM/内置锁与线程池容量治理内容，避免重复堆叠。

## 评估标准（用于自适应推进）

- 完全掌握：能用自己的话讲清原理，并能结合真实代码或线上场景解释
- 基本掌握：核心概念正确，但边界条件或底层关系仍有少量偏差
- 部分理解：会写 `lambda` 或 `Optional`，但讲不清函数式接口、`map`、`flatMap` 的抽象差异
- 初步理解：知道定义，但概念之间还没有串起来，例如把 `ThreadLocal` 和共享同步混在一起
- 尚未理解：把 JVM 内存区、并发工具和异步模型的基本边界讲混
