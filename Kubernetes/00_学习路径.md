# Kubernetes 学习路径与知识图谱

## 学习定位

Kubernetes 不能按“概念合集”来学。它也不应该只分成“工作负载、网络、存储、安全”几个大盒子，因为这样会把最重要的对象藏起来。

比如 Deployment 和 Service：

- Deployment 是你日常发布后端服务时最常碰到的对象，它背后牵出 ReplicaSet、副本控制、滚动发布、回滚、selector、PodTemplate、版本历史和发布卡住时的排障。
- Service 是 Kubernetes 服务发现和稳定访问入口的核心，它背后牵出 ClusterIP、EndpointSlice、selector、kube-proxy、DNS、流量只进 Ready Pod、端口映射和“为什么 Service 存在但没有后端”。

如果把 Deployment 只塞进“工作负载”一节，把 Service 只塞进“网络”一节，学习会变浅。你会知道它们叫什么，但不会真正知道它们在生产里怎么工作、怎么坏、怎么排查。

所以本专题改成“主干原理 + 核心对象专章 + 生产能力专章”的结构。每个核心对象都要单独讲透，而不是一章带过。

## 学完后你应该具备的能力

- 能从单机部署、Docker、Docker Compose 过渡到 Kubernetes，讲清它解决的运行状态管理问题。
- 能理解控制面和工作节点的协作：API Server、etcd、Controller、Scheduler、kubelet、runtime、CNI、CSI 各自负责哪段状态。
- 能深入理解 Pod：为什么它是最小调度单位，为什么不是容器，pause container、namespace、volume、probe、restart policy、termination 都怎么影响线上行为。
- 能把 Deployment 当成一个发布控制系统来理解，而不是只会写 `replicas` 和 `image`。
- 能把 Service 当成动态后端的稳定抽象来理解，而不是只会背 ClusterIP、NodePort、LoadBalancer。
- 能解释 Ingress/Gateway、DNS、CNI、kube-proxy/iptables/IPVS/eBPF、EndpointSlice 之间的链路关系。
- 能理解 ConfigMap、Secret、Volume、PV、PVC、StorageClass 为什么要拆开，以及它们的生命周期差异。
- 能理解 StatefulSet、DaemonSet、Job/CronJob 的使用边界。
- 能从生产视角理解调度、资源、发布、扩缩容、可观测性、安全、权限和故障排查。

## 为什么要这样拆

因为 Kubernetes 的学习主线不是“知识点覆盖”，而是“对象为什么存在，以及对象之间如何协作”。

Deployment、Service、Pod 这三个对象不能合并讲：

- Pod 解决的是“一个可调度运行单元是什么”。
- Deployment 解决的是“如何长期维持和发布一组无状态 Pod”。
- Service 解决的是“如何给一组不断变化的 Ready Pod 提供稳定访问入口”。

它们在真实系统里连在一起，但职责完全不同。合在一章会导致每个都只能浅尝辄止；拆开之后，才能讲 selector、ownerReferences、EndpointSlice、probe、rollout、Service 转发这些关键机制。

## 学习方法建议

- 每学一个对象，都问三个问题：它的 `spec` 表达什么期望？它的 `status` 反映什么真实状态？哪个控制器在调谐它？
- 看 YAML 时不要只看字段，要看对象关系：谁创建谁，谁选择谁，谁更新谁的 status。
- 学 Deployment 时，要画发布时间线；学 Service 时，要画流量链路；学 Pod 时，要画容器、网络命名空间和卷的关系。
- 每章都要带着排障问题学：这个对象最常坏在哪里？`describe` 的事件会怎么写？该看哪个 controller 或 node 组件？
- 不要急着上 Helm、Istio、Operator。先把 Pod、Deployment、Service、ConfigMap、PVC 这些基础对象吃透，后面的生态才不会变成黑盒叠黑盒。

## 学完后你应该能回答的问题

- `kubectl apply` 成功后，Deployment 到 Pod Ready 中间发生了什么？
- Pod 为什么不是容器？为什么一个 Pod 里多个容器共享 IP？
- Deployment 如何通过 ReplicaSet 管理副本和版本？
- Service 为什么能给动态 Pod 提供稳定入口？EndpointSlice 什么时候为空？
- Ingress 和 Service 的边界是什么？Ingress Controller 到底做什么？
- ConfigMap 和 Secret 如何注入容器？为什么 Secret 不是天然保险箱？
- PVC 为什么能让数据生命周期独立于 Pod？为什么它不等于数据库高可用？
- requests、limits、QoS、驱逐之间是什么关系？
- StatefulSet 解决了哪些有状态部署问题，又不能解决哪些数据一致性问题？
- Pending、ImagePullBackOff、CrashLoopBackOff、Service 502 应该分别从哪里查？
