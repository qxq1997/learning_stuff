## 后续扩展方向

- Helm：把一组 Kubernetes YAML 参数化、模板化、版本化。
- Kustomize：用 overlay 管理多环境差异。
- GitOps：用 Git 作为期望状态来源，让集群持续同步。
- Operator：把复杂运维逻辑写成控制器。
- Service Mesh：把服务治理能力下沉到基础设施层，与本仓库 `Service Mesh/` 专题衔接。
