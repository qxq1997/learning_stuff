# Machine Coding 学习路径

## 这个专题要把你带到什么水平

Machine Coding（机器编码轮 / 现场低代码量系统实现轮）是很多公司——尤其是产品型公司和印度系大厂（Flipkart、Swiggy、Uber、Atlassian 等）——单独拉出来的一轮编码面试。它和 LeetCode 算法轮、和系统设计轮都不一样：**给你一个小而真实的需求，限时 60～120 分钟，现场写出一套能编译、能跑、别人愿意接手维护的面向对象代码。**

学完这个专题，你应该能做到：

- 一眼分清眼前这轮到底是 DSA、Machine Coding（LLD）还是 System Design（HLD），并切换到对应的答题模式，而不是把停车场题答成分布式容量估算
- 拿到任意一道 machine coding 题，都能用一套固定的「七步法」从需求澄清推进到可运行代码，而不是上来就闷头写
- 知道面试官那张评分表上到底有哪几栏（能跑、需求覆盖、职责单一、可扩展、模式恰当、可读、边界、并发、可演示），并且在写代码时**主动**去命中每一栏
- 能在该用策略 / 工厂 / 状态 / 观察者的地方恰当地用，而不是为了秀模式而过度设计——machine coding 扣分最狠的两端是「一个 God class 全塞」和「五个接口包一个 if」
- 写出来的代码自带一个 `main`/`Driver`，能当着面试官的面跑出可观察的输出，证明它真的工作
- 能一边写一边讲取舍，并扛得住「现在要加一种新车型 / 新计费方式，你要改哪几个文件」这类扩展性追问

## 重要说明

- 本专题以 **Java** 为主语言（贴合你的主栈），但方法论与语言无关，C++/Go/Python 同样适用；关键差异（并发原语、集合类）会单独点出
- 例题都是 machine coding 的**经典原题**（停车场、Splitwise、电梯、限流器、图书馆等），不是某公司内部实现，用于训练「需求 → 建模 → 可扩展代码」这条肌肉记忆
- 这一轮的核心矛盾永远是**时间 vs 完整度**：不是写得越多越好，而是在限定时间内交出一个跑得通、扩展点清晰、能讲清楚的最小可用系统。整套专题都围绕这个矛盾设计

## 知识图谱（按四层理解）

### 1. 面试轮定位层

- machine coding 到底考什么、为什么大厂要把它从算法轮里单拎出来
- 它和 DSA、System Design 在时长、产出物、考察维度、规模上的根本区别
- 面试官的评分 rubric 拆成一张可执行的检查清单
- 高频致命失分点：跑不起来、逻辑全塞 `main`、God class、过度设计、没有 demo、纠结算法耗光时间、全程不出声

### 2. 解题方法层

- 一套可复用的**七步法**：需求澄清 → 名词建实体 → 动词归方法 → 找变化点抽接口 → 设计门面 API → 按依赖顺序编码 → 写 driver 自测
- 90 分钟的时间预算怎么切（需求 5 / 建模 10 / 确认 5 / 编码 50 / demo+自测 10 / buffer 10）
- 「先跑通主流程，再让它好扩展，最后才谈优化」的推进纪律

### 3. OO 与模式落地层

- 实体 vs 值对象、贫血模型 vs 充血模型、Manager/Service 门面怎么放
- SOLID 在 machine coding 里的具体落地动作（不是背定义，是「哪些维度会变就抽接口」）
- machine coding 高频设计模式地图：策略（计费/分配/拆分/支付）、工厂（造车/造券）、状态（订单/电梯/售货机）、观察者（通知）、单例（系统门面，慎用）
- 内存数据结构与线程安全的默认选择：`Map` / `ConcurrentHashMap`、`synchronized` vs 显式锁、自定义异常 vs 返回码

### 4. 面试表达与演示层

- 怎么写 driver，怎么用一段 happy path + 两个边界把系统「演」出来
- 一边写一边讲：开场怎么对齐范围，中途怎么报告取舍，结尾怎么主动讲扩展点
- 扩展性追问的标准应答套路：「加新 X 我只动 Y，因为变化点被收敛在接口 Z 后面」

## 课程安排

1. `01_machine coding 是什么，凭什么单独拉一轮.md`
   定位、三类编码面试对比、评分 rubric、致命失分点、时间预算
2. `02_通用解题框架：90 分钟从需求到可运行代码.md`
   七步法逐步拆解 + SOLID 落地动作 + 内存数据结构与线程安全默认选择
3. `02b_machine coding 高频设计模式速查.md`
   「触发信号 → 该上哪个模式」速查表 + 10 个高频模式最小骨架 + 每题模式地图 + 反过度设计
4. `03_实战：停车场系统（Parking Lot）.md`
   用七步法完整推一道经典题，给出可编译的 Java，演示策略/工厂/门面与 OCP

5. `04_实战：Splitwise 分账系统.md`
   账本建模（两两 vs 净额）+ 策略拆分（均摊/精确/百分比）+ 金额精度 + 债务简化扩展
6. `05_实战：电梯调度系统.md`
   LOOK 调度策略 + 多电梯派单 + 核心岔路「状态机 ≠ 必须用 State 模式」的取舍
7. `06_实战：限流器（Rate Limiter）.md`
   令牌桶/滑窗策略 + 惰性补充 + 锁粒度 + 多线程压测证明 + 分布式（Redis）扩展

后续按你的节奏继续铺（规划中）：

8. `07_常见反模式与时间管理复盘` —— 把 03～06 提炼成失分点 checklist + 砍需求预案

## 推进规则

每章末尾有「检查站」，先用自己的话讲清楚再看下一章；实战章要求**真的把代码敲进 IDE 跑一遍**——machine coding 这一轮，看懂和写得出来之间的差距，就是这一轮的全部。
