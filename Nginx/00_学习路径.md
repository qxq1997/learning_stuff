# Nginx 学习路径与知识图谱

## 学习定位

Nginx 不是一门“背配置”的技术，而是一套把 Web 服务、反向代理、负载均衡、HTTPS、安全、性能优化和高可用串起来的系统能力。

很多人学 Nginx 时容易走两个极端：

- 只会抄配置，知道 `proxy_pass`、`location`、`rewrite` 怎么写，但说不清为什么这样写。
- 只会背原理，知道 epoll、事件驱动、master-worker，却一碰到真实配置和线上排障就发懵。

这条学习路径的目标，是把你带到一个更完整的位置：

- 你能从浏览器发请求一直讲到 Nginx 怎么接、怎么转、怎么回。
- 你能解释 Nginx 为什么快，而不是只会说“它是异步非阻塞”。
- 你能真正理解 `server`、`location`、`upstream`、`proxy_pass` 这些配置背后的行为。
- 你能把性能优化、安全、HTTPS、高可用和面试表达串成一张完整地图。
- 你不仅会用，还能在面试里讲出设计取舍、常见坑和排障思路。

## 知识图谱（从零到专家）

1. 先建立整体认知
   - Nginx 到底是什么
   - Web 服务器、反向代理、网关、负载均衡分别是什么
   - Nginx 在整体架构中的位置
2. 再理解核心架构
   - master-worker 模型
   - 事件驱动
   - 非阻塞 I/O
   - epoll、accept、惊群
3. 配置体系与请求匹配
   - main、events、http、server、location
   - 虚拟主机
   - location 匹配优先级
   - rewrite、变量、root 与 alias
4. 反向代理与负载均衡
   - `proxy_pass`
   - 常见转发头
   - upstream
   - 负载均衡算法
   - 健康检查与超时
5. 性能优化
   - sendfile、零拷贝
   - keepalive
   - gzip
   - 缓存
   - worker 参数与内核参数
6. HTTPS 与安全
   - TLS 握手
   - 证书链
   - HTTP/2
   - 限流、限连、防盗链、安全响应头
7. 高可用与工程落地
   - 热部署与平滑重载
   - 主备与 VIP
   - 云负载均衡
   - 日志、监控、告警
   - 面试答题框架

## 为什么这样排序

这条路线故意不是“先背配置大全”，而是先建立问题意识。

因为如果你没有先理解：

- Nginx 为什么存在
- 它在请求链路中的位置
- 它为什么扛得住高并发

那么后面的配置项就会变成一堆零碎的语法糖。你会记得 `location ^~` 比 `~` 优先，但不知道这条规则在解决什么问题；你会抄 `proxy_set_header`，但不知道为什么后端会看不到真实客户端 IP。

所以顺序必须是：

1. 先看 Nginx 在解决什么问题
2. 再看它为什么能高性能
3. 再看它如何通过配置表达行为
4. 然后进入反向代理、负载均衡和线上优化
5. 最后再进入 HTTPS、高可用和面试表达

## 学习方法建议

- 学每一课时都先回答一个问题：它在解决什么现实问题。
- 每遇到一个概念，都尽量把它放回真实请求链路里理解，而不是孤立背定义。
- 学配置时，不只看“怎么写”，要同时问“匹配顺序是什么”“默认行为是什么”“容易踩什么坑”。
- 学性能优化时，不要只记“开哪个开关”，更要弄清楚背后的资源模型：CPU、内存、连接、磁盘、网络到底谁是瓶颈。
- 学安全与高可用时，要带着线上故障视角：如果证书报错、502 激增、后端雪崩，我该怎么定位。

## 学完后你应该能回答的问题

- 为什么 Nginx 比传统线程模型更适合高并发场景。
- 一个请求进入 Nginx 后，`server` 和 `location` 是怎么一步步匹配出来的。
- `proxy_pass` 带 `/` 和不带 `/` 到底差在哪里。
- `sendfile`、`keepalive`、`gzip`、缓存分别优化的是哪一段链路。
- 证书链为什么会不完整，HTTPS 报错通常从哪里查起。
- 热重载为什么能不中断服务，主备切换的本质是什么。
