# Python 学习路径与知识图谱

## 学习定位

如果只是把 Python 学成“会写脚本的工具语言”，那它在面试里很容易显得浅。

因为真正拉开差距的，往往不是会不会写 `for` 循环，而是下面这些问题你能不能讲透：

- Python 这门语言和 `CPython` 这个实现到底是什么关系
- 为什么 Python 常被叫做解释型语言，但它其实也有编译阶段
- 为什么说“变量不是盒子，而是名字绑定到对象”
- `list`、`dict`、`set` 为什么快，代价又是什么
- GIL 到底锁住了什么，为什么线程不总能加速
- `asyncio` 为什么不是“更轻量的线程”
- 引用计数、垃圾回收、描述符、MRO、闭包这些概念在工程里到底有什么实际价值

你已经有后端开发经验，所以这条路线不会停留在“语法速成”层面，而是按下面这条主线来组织：

**语言本质 -> 数据模型 -> 运行时与底层机制 -> 工程实践 -> 并发与网络 -> 性能与面试表达**

目标不是把 Python 题库背一遍，而是建立一套系统判断力。学完之后，你应该既能写得出来，也能解释为什么这么写，以及它在底层到底发生了什么。

## 知识图谱（从语言到运行时，再到工程）

1. 语言本质与执行模型
   - Python 语言规范与 `CPython` / `PyPy`
   - 源码、字节码、解释器循环
   - 动态类型、强类型、鸭子类型
   - 名字绑定、作用域、命名空间
2. 数据模型与核心容器
   - 对象、类型、身份、值
   - `list`、`tuple`、`dict`、`set`
   - 哈希、扩容、时间复杂度
   - 可变与不可变、浅拷贝与深拷贝
3. 函数式抽象与协议
   - 函数是一等公民
   - 参数绑定
   - 闭包
   - 装饰器
   - 迭代协议、生成器、上下文管理器
4. 面向对象与高级特性
   - 类与实例
   - 方法解析顺序（MRO）
   - 描述符
   - `property`
   - `dataclass`
   - 元类
5. 运行时与内存管理
   - `PyObject`
   - 引用计数
   - 循环引用
   - `gc`
   - 小整数缓存与字符串驻留
   - 栈帧与调用开销
6. 模块化与工程实践
   - `import` 系统
   - 包、模块、命名空间
   - 虚拟环境
   - 依赖管理
   - 异常处理
   - 日志
   - 测试
7. 并发与网络编程
   - 线程、多进程、协程
   - GIL
   - `asyncio`
   - 线程池、进程池
   - `socket`
   - HTTP 客户端
   - WSGI / ASGI
8. 性能与面试表达
   - 复杂度与数据结构选型
   - profiling
   - 内存分析
   - 常见性能陷阱
   - 高频 Python 面试题答题框架

## 为什么这样排序

很多人学 Python 的顺序是：

- 先学语法糖
- 再背几个高频八股
- 最后临时抱佛脚去记 GIL、协程和 `dict` 原理

这种路径有一个明显问题：知识点彼此断开。

比如你可能知道：

- `dict` 平均查找是 `O(1)`
- Python 是动态类型语言
- 闭包里会发生“晚绑定”
- 多线程受 GIL 影响

但你却说不清这些现象背后共同的运行时前提是什么。于是知识会变成零碎结论，而不是一套体系。

这次我们反过来走：

1. 先搞清 Python 到底是什么，它的执行模型和对象模型是什么
2. 再进入变量、容器、函数、闭包、迭代协议这些语言基础设施
3. 然后再看面向对象、描述符、内存管理、`import` 和工程写法
4. 最后进入线程、协程、多进程、Web 栈、性能优化和面试表达

这样一来，后面很多“背诵题”都会自动连起来：

- 你会知道为什么“变量不是盒子”
- 你会知道为什么 `dict` 快但吃内存
- 你会知道为什么生成器能省内存
- 你会知道为什么 `asyncio` 不是并行而是协作式调度
- 你会知道为什么 Python 代码常常开发效率高、运行效率却未必高

另外，原来已经有一篇多线程专题文章。现在它会保留为第 12 课，等前面的对象模型、运行时、并发全景和协程地基铺好之后再读，理解会稳很多。

## 学习方法建议

- 每学一个知识点，都追问三个问题：它解决什么问题、底层怎么做、代价是什么。
- 明确区分“语言层结论”和“`CPython` 实现细节”，不要把两者混成一团。
- 对高频概念尽量做到“可画图解释”，比如名字绑定、闭包、事件循环、引用计数。
- 不只记结论，要写最小实验去验证，比如可变对象共享、浅拷贝、生成器惰性求值、线程切换。
- 面试表达时尽量按“定义 -> 原理 -> 代价 -> 适用场景 -> 常见误区”这个顺序来答。

## 学完后你应该能回答的问题

- Python 这门语言和 `CPython` 解释器到底是什么关系？
- 为什么说“Python 是解释型语言”这句话不够准确？
- 为什么 Python 里变量更准确地说是“名字绑定到对象”，而不是“变量里装着值”？
- `list`、`tuple`、`dict`、`set` 的核心底层结构和复杂度各是什么？
- 闭包为什么会出现晚绑定问题？装饰器为什么本质上是高阶函数？
- 描述符和 `property` 为什么能改变属性访问行为？
- Python 的引用计数和垃圾回收是怎么配合工作的？
- GIL 的真实影响到底是什么，哪些场景下线程仍然非常有用？
- 协程和线程的根本差异是什么？`asyncio` 为什么适合高并发 I/O？
- 如果面试官问“Python 为什么慢”，你能否给出一套结构化回答？
