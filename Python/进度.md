## 当前学习假设

- 你有后端开发经验，但默认不假设你已经系统梳理过 Python 的并发模型。
- 学习目标不只是“会写脚本”，而是把线程、协程、多进程和工程实践真正用对地方。
- 教学会优先贴近真实开发场景，例如批处理、接口聚合、爬虫、后台任务和 Web 服务。

## 评估标准（用于自适应推进）

- 完全掌握：能根据任务特征选对并发模型，并解释背后的原理和取舍
- 基本掌握：知道线程池、多进程、协程的大致适用范围，但工程细节仍需补强
- 部分理解：知道 GIL、多线程、多进程这些词，但还不能稳定做场景判断
- 尚未理解：把 I/O 密集、CPU 密集、线程、协程、多进程的边界混在一起
