# RAG 学习进度（2026 面试导向版）

## 学习主题

RAG（Retrieval-Augmented Generation）— 面向 2026 年 AI 应用后端 / 搜索知识库方向岗位。

## 学习者画像

- 背景：5 年经验后端开发工程师，熟悉数据库、缓存、消息队列、异步任务、分布式系统。
- 目标：具备从系统设计面试、工程落地、技术选型三个层面讨论 RAG 的能力。
- 当前诉求：不要 Demo 级别内容，要业界最新进展 + 历史演化 + 工程权衡。
- 代码偏好：Python。
- 学习重点：数据层、检索层、高级架构（Agentic / Graph / 多模态）、评测、工程落地。

## 本轮大纲重写原因

旧版（01 + 02 + 02b）深度不够，缺乏 2024-2026 演化视角。本轮重构为 16 章 + 3 附录，覆盖从问题本质到面试系统设计的完整链路。详见 `00_学习路径.md`。

3. 每章标准结构：学习目标 / 问题背景 / 演化 / 原理 / 代码 / 工程权衡 / 面试 / 小结。
4. 代码全 Python，尽量可跑。
