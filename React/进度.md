## 当前学习假设

- 你是后端开发工程师，默认熟悉接口、状态、模块拆分和工程化，但不默认系统学过现代前端。
- 目标不是只会写几个组件，而是能理解 React 的设计取舍，并能写出可维护的业务页面。
- 课程会尽量贴近真实工作：后台列表页、搜索筛选、分页、详情页、编辑弹窗、权限按钮、接口错误处理、表单提交这些场景。

## 评估标准（用于自适应推进）

- 完全掌握：能用状态驱动 UI 的视角解释组件设计，并能把一个业务页面拆成合理组件。
- 基本掌握：知道 React 大方向，但对重新渲染、State 归属、Effect 边界还不够稳。
- 部分理解：会写 JSX 和简单事件，但主要靠模仿，不知道为什么这么写。
- 尚未理解：仍然把 React 当成“操作 DOM 的另一种语法”，或认为重新渲染就是刷新整个页面。
