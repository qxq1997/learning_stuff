## 当前学习假设

- 你是后端开发工程师，平时会写业务 SQL，但默认不假设你系统学过数据库执行原理。
- 目标不是成为 DBA，而是能对高频慢 SQL、分页、索引失效、聚合查询、线上慢日志做靠谱判断。
- 学习优先贴近真实工作：接口列表、后台任务、账单查询、订单明细、统计报表、日志检索这类典型后端场景。

## 评估标准（用于自适应推进）

- 完全掌握：能结合执行计划解释 SQL 为什么慢，并提出合理优化方案
- 基本掌握：知道大方向，但对回表、排序、临时表、索引选择的细节还不够稳
- 部分理解：会背一些优化建议，但无法说明“为什么”
- 尚未理解：把所有慢 SQL 都简单归因于“没加索引”或“数据库不行”
