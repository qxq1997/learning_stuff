# 系统设计总览

本文档描述 TalentHub 的第一版系统目标、核心业务流程和总体架构。

## 1. 产品目标

TalentHub 不是一个通用问答系统，而是一个围绕企业培训构建的业务平台。系统需要完成如下闭环：

1. 导入企业内部文档，形成可检索知识库
2. 基于知识库和知识点生成题目
3. 将题目沉淀为可维护题库
4. 基于题库进行组卷和考试分发
5. 用户提交考试后完成自动判卷与 AI 反馈
6. 追踪学习进度、能力趋势和组织侧指标

## 2. 目标用户

- 普通员工：参与学习、答题、查看反馈、跟踪成长路径
- 题库管理员：管理知识库、审查题目、维护试卷
- 培训管理员：配置学习路径、安排考试、查看培训结果
- 管理者：查看团队能力趋势和平台指标

## 3. 核心能力

### 3.1 知识库

- 上传企业内部文档
- 对文档做切片、向量化和检索
- 为题目生成和问答提供上下文

### 3.2 题库

- 根据知识点、题型、难度生成题目
- 保存题目版本、来源文档和解析
- 支持人工审核和维护

### 3.3 考试

- 组卷
- 分发考试
- 提交答案
- 记录考试状态和分数

### 3.4 评估

- 客观题自动判分
- 主观题基于评分规则和 AI 模型评分
- 输出逐题反馈与薄弱点分析

### 3.5 学习与分析

- 记录学习路径和完成情况
- 输出个人成长趋势
- 汇总组织与平台级指标

## 4. 总体架构

第一阶段采用模块化单体。

这样做的原因是：

- 项目目前从 0 开始，先保证认知一致和交付速度
- 模块边界可以先在代码内清晰建立
- 后续如果某个模块独立增长，再拆成服务也更容易

## 5. 推荐的本地部署技术栈

- 前端：React + TypeScript + Ant Design
- 后端：FastAPI
- 数据库：PostgreSQL
- 向量检索：Qdrant
- 缓存：Redis
- 任务队列：Celery 或 Dramatiq
- 文件存储：本地文件系统，必要时兼容 MinIO

## 6. AI 能力的位置

AI 只放在明确的业务节点里：

- 生成题目
- 生成解析
- 主观题评分
- 生成逐题反馈
- 生成学习建议
- 基于知识库做检索增强

系统的主业务流仍由确定性的应用服务控制，不交给 Agent 自由编排。

## 7. 第一阶段不优先做的内容

- 多 Agent 协作架构
- 复杂模型路由策略
- LoRA / SFT / 后训练
- 微服务拆分
- 大量兼容性 fallback

这些能力只有在第一版业务闭环稳定后才考虑引入。
