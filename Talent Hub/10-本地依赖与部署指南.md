## 2. 方式一：直接在本机运行

### 2.1 创建虚拟环境

```bash
make backend-install
```

如果你要在本机运行 backend，并且希望“网页 URL 出题”支持 Docsify、React、Vue 这类前端渲染页面，还需要额外安装 Playwright 的浏览器：

```bash
make backend-browser-install
```

如果不使用 `make`，也可以手动执行：

```bash
python3 -m venv backend/.venv
backend/.venv/bin/pip install -r backend/requirements-dev.txt
backend/.venv/bin/playwright install chromium
```

### 2.2 配置环境变量

```bash
cp backend/.env.example backend/.env
```

当前这台机器上我已经为你创建了本地可用的 [backend/.env](/Users/xinqi/WebstormProjects/TalentHub/backend/.env)。

当前还需要注意两个后端配置项：

- `DOCUMENT_STORAGE_ROOT`
  - 知识文档原始文本的本地持久化目录
- `DEVELOPMENT_OPERATOR_EMAIL`
  - 在未接入登录前，手动导入文档时使用的开发管理员邮箱
- `LLM_PROVIDER`
  - 是否启用 OpenAI 题目生成预览
- `OPENAI_API_KEY_FILE`
  - OpenAI API Key 文件路径；文件内容必须是原始 key 本身，不要再包一层 `OPENAI_API_KEY=...`
- `DEEPSEEK_API_KEY_FILE`
  - DeepSeek API Key 文件路径；和 OpenAI 一样，只放原始 key 本身
- `WEB_PAGE_RENDER_TIMEOUT_MS`
  - 网页渲染等待上限，单位毫秒
- `WEB_PAGE_MIN_CONTENT_CHARS`
  - 网页正文最小字符数；低于这个阈值会直接失败
- `WEB_PAGE_MAX_NESTED_LINK_CANDIDATES`
  - 单次最多展示多少个同域子页供用户确认范围
- `WEB_PAGE_MAX_IMAGE_CANDIDATES`
  - 单次最多提取多少张网页配图的图片线索
- `QUESTION_BANK_TAG_CACHE_TTL_SECONDS`
  - 题库标签列表缓存 TTL，单位秒
- `WEB_PAGE_FETCH_CACHE_TTL_SECONDS`
  - 网页抓取结果缓存 TTL，单位秒

当前约定：

- `backend/.env`
  - 本地运行 FastAPI 时直接读取
  - Docker Compose 启动 backend 时也会通过 `env_file` 读取

也就是说，你现在只需要维护这一份本地后端配置，不需要再额外在 shell 里重复导出同样的 OpenAI 变量。

对于 OpenAI 密钥，本项目当前约定不把密钥明文写进 `backend/.env`。正确方式是：

- 把真实 key 单独放在宿主机文件里
- `backend/.env` 只写各 provider 的 `*_API_KEY_FILE=...`
- Docker backend 通过只读挂载去读取这些文件

### 2.3 启动 PostgreSQL 和 Redis

你可以本机安装，也可以先用 Docker 跑基础设施：

```bash
docker compose -f docker-compose.local.yml up -d postgres redis qdrant minio
```

### 2.4 初始化数据库

如果你不是通过 Docker 首次初始化 PostgreSQL，请手动执行：

```bash
psql postgresql://talenthub:talenthub@localhost:5432/talenthub -f backend/sql/001_init_schema.sql
```

注意：

- `001_init_schema.sql` 是初始化脚本，默认只用于空数据库
- 该脚本不追求可重复执行；如果重复执行失败，这是符合当前 `fail fast` 原则的
- 后续数据库演进应通过 Alembic 迁移完成

如果你要使用“知识库手动导入”功能，还需要补本地开发管理员种子：

```bash
make db-seed-dev
```

这个种子会创建：

- 开发部门 `TalentHub Dev`
- 角色 `system_admin`
- 角色 `department_admin`
- 角色 `employee`
- 平台管理员 `admin@talenthub.local`
- 开发部门管理员 `dept-admin@talenthub.local`
- 开发部门成员 `member@talenthub.local`
- 其他部门管理员 `other-admin@talenthub.local`
- 其他部门成员 `other-member@talenthub.local`

### 2.5 启动后端

```bash
make backend-run
```

注意：

- 如果本机 backend 没执行过 `make backend-browser-install`，网页 URL 出题会因为缺少 Chromium 而失败
- 这是预期的 fail-fast 行为，不会偷偷降级成静态抓取

启动后可访问：

- 健康检查：`http://localhost:8000/api/v1/healthz`
- OpenAPI：`http://localhost:8000/docs`

### 2.6 启动前端

```bash
cp frontend/.env.example frontend/.env
make frontend-install
make frontend-dev
```

启动后默认访问：

- `http://localhost:5173`

当前前端已经接入正式登录/注册页。执行开发种子后，可以直接使用：

- `admin@talenthub.local / talenthub123`

登录；前端会保存 Bearer token，后端按 token 识别当前用户、部门和角色。

## 3. 方式二：使用 Docker Compose 运行

当前仓库已经提供：

- [docker-compose.local.yml](/Users/xinqi/WebstormProjects/TalentHub/docker-compose.local.yml)
- [backend/Dockerfile](/Users/xinqi/WebstormProjects/TalentHub/backend/Dockerfile)

启动：

```bash
docker compose -f docker-compose.local.yml up --build
```

或后台启动：

```bash
docker compose -f docker-compose.local.yml up -d --build
```

这会启动：

- PostgreSQL
- Redis
- MinIO
- FastAPI backend

当前 Docker backend 在构建镜像时会自动安装 Playwright Chromium，所以容器内的网页 URL 出题不需要额外手工安装浏览器。

### 3.1 当前推荐模式

当前推荐直接使用 Docker Compose 托管基础设施：

- Docker PostgreSQL
- Docker Redis
- 可选 Docker MinIO
- 本地运行 FastAPI，或后续也切到 Docker

如果你现在要让 Docker backend 使用 OpenAI，只需要先确认 [backend/.env](/Users/xinqi/WebstormProjects/TalentHub/backend/.env) 里已经写好：

- `LLM_PROVIDER=openai`
- `OPENAI_API_KEY_FILE=...`
- `OPENAI_MODEL_NAME=gpt-5-mini`
- `QUESTION_BANK_TAG_CACHE_TTL_SECONDS=300`
- `WEB_PAGE_FETCH_CACHE_TTL_SECONDS=900`

只启动 PostgreSQL：

```bash
make db-up
```

对空数据库执行首次 schema 初始化：

```bash
make db-bootstrap
make db-stamp
make db-seed-dev
```

只启动 Redis：

```bash
make redis-up
```

同时拉起 PostgreSQL、Redis 和 MinIO：

```bash
make infra-up
```

### 3.2 端口说明

- Docker PostgreSQL：`5432`
- Redis：`6379`
- MinIO API：`9000`
- MinIO Console：`9001`
- Backend API：`8000`

### 3.3 本地持久化目录

当前 Docker Compose 会把以下目录持久化到宿主机：

- `postgres_data/`
- `redis_data/`
- `minio_data/`
- `storage_data/`

其中：

- `storage_data/` 用于保存知识库手动导入的原始文本文件

## 4. 当前部署说明

### 4.1 这是一个本地优先项目

第一阶段默认部署方式是：

- 单机
- 本地数据库
- 本地 Redis
- 本地对象存储
- 本地或局域网模型服务

### 4.2 当前后端容器的模型地址

`docker-compose.local.yml` 里默认把模型地址写成：

`http://host.docker.internal:11434`

这适用于宿主机上运行 Ollama 的场景。如果你的模型服务不在宿主机，需要改成实际地址。
