# 前端架构与页面规划

本文档说明 TalentHub 当前前端的定位、技术选型和下一阶段页面建设顺序。

## 1. 当前状态

项目原先只有后端、数据库和基础设施骨架，没有真正的前端目录。

当前已经补齐前端工作台，并完成了第一条读链路和第一条写链路：

- [frontend/package.json](/Users/xinqi/WebstormProjects/TalentHub/frontend/package.json)
- [frontend/vite.config.ts](/Users/xinqi/WebstormProjects/TalentHub/frontend/vite.config.ts)
- [frontend/src/App.tsx](/Users/xinqi/WebstormProjects/TalentHub/frontend/src/App.tsx)
- [frontend/src/lib/api.ts](/Users/xinqi/WebstormProjects/TalentHub/frontend/src/lib/api.ts)

这版前端当前已经具备：

- React + TypeScript + Vite 工作台
- 基础视觉语言
- 与后端 `/api/v1/healthz` 的联调能力
- 与后端 `/api/v1/knowledge/documents` 的分页查询联调
- 与后端 `/api/v1/knowledge/documents/manual-imports` 的文本文档创建联调
- 与后端 `/api/v1/question-bank/generation-previews` 的 AI 题目预览联调
- 与后端 `/api/v1/exams/papers` 的试卷创建、列表和详情联调

## 2. 技术选型

- React
- TypeScript
- Vite
- Ant Design

原因：

- 适合内部平台型页面
- 表格、表单、抽屉、布局和数据录入能力成熟
- 与后端 REST API 协作简单

## 3. 当前页面

当前前端已经拆成多页面工作台：

1. 总览页
2. 出题工作台
3. 知识库页面
4. 题库页面
5. 考试页面

## 4. 页面建设顺序

建议按下面顺序推进：

### 4.1 知识库工作台

- 已完成：文档状态列表
- 已完成：手动导入文本文档
- 下一步：真正的文件上传
- 下一步：文档切片预览
- 下一步：检索结果查看

### 4.2 题目生成面板

- 选择知识点
- 选择题型和难度
- 发起 AI 出题任务
- 查看生成结果并入库

### 4.3 试卷与考试页面

- 已完成：按标签筛题
- 已完成：显式勾选题目组卷
- 已完成：试卷列表与试卷详情
- 已完成：创建测试分配并开始答题
- 已完成：提交并查看 AI 判卷结果
- 下一步：正式分配列表
- 下一步：个人考试历史页

### 4.4 学习与分析页面

- 个人学习路径
- 个人考试趋势
- 团队指标
- 平台使用指标

## 5. 前后端联调规则

前端通过：

- `VITE_API_BASE_URL`

来确定 API 地址。

当前该变量是必须项，不提供就直接报错，不做隐式 fallback。

## 6. CORS 说明

后端已经允许以下开发域名：

- `http://localhost:5173`
- `http://127.0.0.1:5173`

- [backend/app/main.py](/Users/xinqi/WebstormProjects/TalentHub/backend/app/main.py)
- [backend/app/shared/config/settings.py](/Users/xinqi/WebstormProjects/TalentHub/backend/app/shared/config/settings.py)

## 7. 启动方式

前端依赖 Node.js。当前这台机器上已经完成依赖安装并通过构建验证。

可执行：

```bash
cp frontend/.env.example frontend/.env
make frontend-install
make frontend-dev
```

默认开发地址：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8000`

## 8. 当前原则

- 先让页面成为真实业务工作台，不做假大空演示站
- 页面状态和后端状态显式可见
- 环境缺失时直接失败，不做偷偷回退
- 先把查询和创建链路做真，再继续扩展上传、详情、解析和任务编排
