# JS 渲染网页抓取设计

## 背景

之前网页抓取只读取静态 HTML。

这对普通文章页可以工作，但对这些页面会失败：

- Docsify 文档站
- React 单页应用
- Vue 单页应用
- 使用 `#` hash 路由的知识站点

这类页面真正的正文通常不是服务端直接返回的，而是浏览器加载脚本后才渲染出来。

## 这次改动的核心判断

TalentHub 的“网页 URL 出题”不再走“先静态抓一下，不行再想别的”的补丁式逻辑。  
现在它采用一条明确的流程：

1. 启动无头 Chromium
2. 打开目标 URL
3. 等待正文区域出现足够长度的文本
4. 读取渲染后的 DOM
5. 提取正文
6. 正文仍然不合格就直接报错

这符合项目的 `fail fast` 原则。

## 当前技术实现

实现位置：

- [backend/app/modules/question_bank/infrastructure/web_fetcher.py](/Users/xinqi/WebstormProjects/TalentHub/backend/app/modules/question_bank/infrastructure/web_fetcher.py)

当前使用：

- `playwright`
- `chromium`
- `BeautifulSoup`

抓取流程：

1. `page.goto(url, wait_until="domcontentloaded")`
2. 等待 `article` / `main` / `body` 的 `innerText` 长度达到阈值
3. 读取 `page.title()` 和 `page.content()`
4. 用 `BeautifulSoup` 清理非正文标签
5. 从 `article`，否则 `main`，否则 `body` 抽正文
6. 标题优先取 `h1`
7. 对 `#/...` 这类 hash 路由页面，若没有 `h1`，先尝试从 URL 片段里提取章节名
8. 再尝试正文第一行
9. 最后才回退到浏览器标题

## 当前配置项

新增配置：

- `WEB_PAGE_RENDER_TIMEOUT_MS`
- `WEB_PAGE_MIN_CONTENT_CHARS`
- `WEB_PAGE_MAX_NESTED_LINK_CANDIDATES`
- `WEB_PAGE_MAX_IMAGE_CANDIDATES`

含义：

- `WEB_PAGE_RENDER_TIMEOUT_MS`
  - 页面渲染等待超时时间
- `WEB_PAGE_MIN_CONTENT_CHARS`
  - 正文最小字符数
- `WEB_PAGE_MAX_NESTED_LINK_CANDIDATES`
  - 单次最多展示多少个可确认的同域子页
- `WEB_PAGE_MAX_IMAGE_CANDIDATES`
  - 单次最多提取多少张网页配图的图片线索

如果渲染后正文长度低于阈值，接口会直接失败。

另外，网页渲染成功后仍然要受题目生成输入上限约束：

- `QUESTION_GENERATION_SOURCE_MAX_CHARS`

当前默认值是：

- `20000`

如果网页正文超过这个上限，系统不会直接开始分页生成，也不会偷偷把超长内容硬截成一段模糊文本。

现在的行为是：

1. 后端先计算总字数和分页数量
2. 返回 `confirmation_required`
3. 前端弹窗告诉用户需要分几页处理
4. 用户点击确认后，后端才真正按页生成题目

## 为什么之前那个 Datawhale 页面会失败

例如这个链接：

- `https://datawhalechina.github.io/hello-agents/#/...`

它属于典型的 Docsify + hash 路由页面。

旧实现里，后端请求 URL 时拿到的只是 HTML 壳页面，正文还没被 JS 渲染出来，所以最终只抽到了非常短的文本。

新实现里，浏览器会先把页面渲染完，再去抽正文，因此这类页面可以被正常处理。

## Docker 与本地运行差异

### Docker backend

`backend/Dockerfile` 在构建时会自动安装：

- Playwright Python 包
- Chromium 浏览器

所以 Docker 模式下不需要额外手动安装浏览器。

### 本地 backend

如果你直接运行：

- `make backend-run`

那么除了 `make backend-install`，还需要执行：

- `make backend-browser-install`

否则网页抓取会因为本机没有 Chromium 而失败。

## 错误语义

网页 URL 出题当前可能返回这些明确错误：

- 页面打开失败
- 页面状态码不是 `200`
- 页面不是 HTML
- 页面标题不存在
- 页面渲染超时
- 渲染后正文长度不足
- 渲染成功但正文超过题目生成输入上限且分段失败
- 检测到同域子页但用户尚未确认抓取范围

这些错误都应该被认为是“网页来源不满足出题条件”，而不是系统自己去猜应该怎么继续。
