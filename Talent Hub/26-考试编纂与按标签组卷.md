# 考试编纂与按标签组卷

本文档说明 TalentHub 当前试卷作者工作台是如何组织的，以及“按标签筛题 -> 创建试卷 -> 立即可用”的实际规则。

## 1. 当前目标

当前试卷编纂阶段解决的是这些问题：

- 用已经生成并沉淀到题库的题目编纂成考试
- 通过标签、状态、关键词筛选候选题
- 显式勾选题目
- 创建试卷并查看试卷详情
- 按 `public / private` 指定考试可见性

也就是说，当前已经实现的是：

- 试卷作者工作台

## 2. 为什么先做“显式勾选”，不做自动抽题

TalentHub 当前继续遵守这条原则：

- fail fast
- 不做过度猜测

所以这版考试编纂不会：

- 自动猜测你想要哪几道题
- 在题不够时偷偷补题
- 自动跨标签扩题

当前规则是：

1. 你先按标签、状态、题型、关键词筛候选题
2. 你自己勾选题目
3. 系统按你的勾选顺序生成试卷

这样做的好处是：

- 试卷边界清晰
- 组卷结果可解释
- 后端验证逻辑简单
- 不会因为“自动补题”把错误悄悄吞掉

## 3. 当前后端接口

### 3.1 试卷列表

- `GET /api/v1/exams/papers`

支持：

- 分页
- 按状态筛选
- 按标签筛选
- 按标题 / 说明 / 标签关键字搜索

可见性规则：

- `public` 已发布试卷对所有账号可见
- `private` 已发布试卷只对本部门可见
- 普通成员也可以查看自己有权限看到的试卷，但不能造卷

### 3.2 试卷详情

- `GET /api/v1/exams/papers/{exam_paper_id}`

返回：

- 试卷基本信息
- 试卷标签
- 题目快照列表
- 每题分值、顺序、题区名称

### 3.3 创建试卷

- `POST /api/v1/exams/papers`

请求中需要显式给出：

- 试卷标题
- 试卷说明
- 试卷可见性
- 考试时长
- 每题分值
- 及格比例
- 题区名称
- 本次筛题使用的标签
- 选中的题目 ID 和题目版本 ID

当前规则：

- 新建试卷始终从 `draft` 开始
- 只有 `department_admin` 或 `system_admin` 可以创建

### 3.4 直接生效规则

当前规则已经改成：

- 有权限的管理员创建试卷后直接可用
- 不再要求先送审再发布
- 试卷仍然保留 `status` 字段，主要用于兼容已有数据和未来扩展

## 4. 关键业务规则

### 4.1 必须至少选择一道题

后端不允许空试卷创建成功。

### 4.2 不允许重复选同一题

同一个 `question_id` 不能在一次创建请求里重复出现。

### 4.3 只接受当前最新题目版本

前端候选题列表展示的是题目的最新版本。

如果你在勾选之后、创建之前，这道题又被编辑出新版本，后端会直接失败，并提示：

- 题目版本已经过期
- 需要刷新候选题列表后重试

这里故意不做隐式升级，是为了避免你以为自己在考 A 版题，系统却偷偷塞进了 B 版。

### 4.4 试卷总分和及格分由后端计算

当前前端只输入：

- 每题分值
- 及格比例

后端统一计算：

- `total_score`
- `pass_score`

这样可以避免前端自己算错后把错误值写进数据库。

### 4.5 试卷会保存“编纂标签”

当前除了把题目快照写进 `exam_paper_questions`，还会把本次组卷时选用的标签写进：

- `exam_paper_tags`

它的作用是：

- 试卷列表按标签过滤
- 后面查这套卷是围绕哪些主题编的
- 为后续“按标签继续补题”留下上下文

### 4.6 私有考试不允许跨部门访问

当前实现对私有考试采用严格部门边界：

- 同部门成员可以查看已发布私有考试
- 其他部门访问时直接失败
- 不做“看不到就返回空”的静默处理

这个规则也会影响：

- 试卷详情查看
- 试卷分配
- 考试尝试

## 5. 前端页面结构

当前已经新增独立的：

- `考试` 页面

页面拆成两块：

### 5.1 左侧：组卷工作台

- 标签筛选
- 题目状态筛选
- 题型筛选
- 关键词搜索
- 候选题表格
- 勾选题目
- 显示已选题数、总分、及格分

### 5.2 右侧：试卷配置

- 标题
- 说明
- 试卷可见性
- 时长
- 每题分值
- 及格比例
- 题区名称

### 5.3 下方：已创建试卷

- 试卷列表分页
- 按状态 / 标签 / 关键字筛选
- 点击查看试卷详情
- 对有权限的用户展示试卷详情和题目内容

## 6. 当前数据库变化

这一阶段的关键持久化对象包括：

- `exam_papers`
- `exam_paper_questions`
- `exam_paper_tags`

新增的重要字段：

- `exam_papers.visibility`
- `exam_papers.department_id`

新增的重要索引：

- `ix_exam_papers_visibility_department_status_updated_at`
- `ix_exam_paper_tags_tag_name_exam_paper_id`

## 7. 当前边界

这一轮已经完成：

- 按标签筛题
- 显式勾选组卷
- 公共 / 私有试卷

仍可继续增强的点：

- 试卷编辑
- 批量分配界面
- 个人考试中心
- 自动组卷策略
