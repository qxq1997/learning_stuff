# 题库与考试查看编辑设计

## 目标

把“题库”和“考试编纂”都补成真正可用的管理界面，而不是只有创建入口。

当前落地目标是：

- 题库中的题目可以查看
- 题库中的题目可以编辑
- 已创建的试卷可以查看
- 已创建的试卷可以重新加载到编纂工作台并编辑

## 题库查看与编辑

### 查看

题库列表页提供独立的“查看”入口。

查看弹窗展示：

- 题型
- 状态
- 难度
- 版本号
- 来源类型
- 来源标题和来源 URL
- 标签
- 题干
- 选项
- 标准答案或参考答案代码
- 解析

代码题在查看时使用只读 CodeMirror 展示参考答案代码，避免长代码块难以阅读。

### 编辑

题库列表页继续保留“编辑”入口。

编辑规则：

- 编辑保存后创建新的 `question_versions`
- 题目主记录保持原 `question_id`
- 标签会被整组替换
- 保存后立即生效，不再额外走人工审核

这条规则的好处是：

- 历史引用仍然稳定
- 当前题库始终指向最新版本
- 题目修改有明确版本轨迹

## 试卷查看与编辑

### 查看

考试页已有“查看”入口，展示：

- 试卷标题、说明、状态、可见性
- 时长、总分、及格分
- 试卷标签
- 题目清单

题目清单会展示：

- 序号
- 题型
- 题干
- 难度
- 分值
- 题目版本
- 题区名称
- 标签

### 编辑

试卷编辑采用“加载到现有编纂工作台”的方式，而不是再做一套完全独立的编辑页。

点击“编辑”后，系统会把这套试卷的内容加载到右侧编纂区：

- 标题
- 说明
- 可见性
- 时长
- 每题分值
- 及格比例
- 题区名称
- 试卷标签
- 当前题目清单

然后用户可以：

- 修改元信息
- 删除已选题
- 从候选题列表里补充新题
- 保存修改

保存时走 `PATCH /api/v1/exams/papers/{exam_paper_id}`。

## 试卷编辑的 fail-fast 规则

为了避免破坏历史考试数据，试卷编辑不是无限制开放。

当前规则非常明确：

- 如果试卷还没有产生任何分配记录，可以编辑
- 如果试卷已经产生分配或作答记录，直接拒绝编辑

拒绝原因：

- 试卷题目清单变更后，会影响后续作答与判卷的一致性
- 已分配试卷再被重写，容易让考试历史和实际展示内容脱节
- 当前项目明确采用 fail-fast，不做隐式兼容和补丁式兜底

因此系统会直接返回错误，而不是偷偷复制试卷、偷偷保留旧题，或者试图猜测你想怎么处理。

如果后续需要“已发布试卷也能继续演进”，更合适的设计不是覆盖编辑，而是：

- 基于现有试卷复制一份新试卷
- 新试卷作为新的可分配版本继续使用

## 前端交互说明

### 题库页

- 列表支持分页
- 操作列提供“查看 / 编辑”
- 查看是只读详情
- 编辑是可写表单

### 考试页

- 列表支持分页
- 操作列提供“查看 / 编辑 / 测试作答”
- 编辑会把试卷加载回编纂区
- 编纂区会显示当前已选题清单，支持移除和清空

## 后端接口

本次涉及的核心接口：

- `GET /api/v1/question-bank/questions/{question_id}`
- `PATCH /api/v1/question-bank/questions/{question_id}`
- `GET /api/v1/exams/papers/{exam_paper_id}`
- `PATCH /api/v1/exams/papers/{exam_paper_id}`

## 已验证结果

当前已经实际验证：

- 题目详情接口可用
- 题目编辑接口可用
- 试卷详情接口可用
- 试卷编辑接口可用
- 试卷在无分配记录时可编辑
- 试卷在已产生分配记录后会被拒绝编辑
