# 考试按标签装题与语义日志增强

## 这次改了什么

这轮改动主要分成两块：

1. 考试编纂页支持按标签一键装题
2. TalentHub 全局补更多业务语义日志，并压低系统启动噪音

## 考试编纂页：按标签一键装题

之前考试页的流程偏手工：

- 先筛题
- 再逐条勾选
- 最后创建试卷

这次补上了更顺手的方式：

- 可以继续手动勾选
- 也可以先填写“装题数量”，再点击“按当前标签装入指定数量”
- 自动装题完成后，仍然可以手动删除或补充
- 同时保留“刷新候选题”

### 一键装题使用的标签来源

当前规则是：

- 如果“候选题筛选”里已经选了标签，就优先使用这里的标签
- 如果这里没选标签，但“试卷标签”已经填了标签，就使用试卷标签
- 两边都没有标签时，不允许一键装题，直接 fail fast 提示用户先选标签

### 一键装题使用的筛选条件

它不只是看标签，还会同时带上当前页面上的这些条件：

- 标签
- 题目状态
- 题型
- 关键字

这样逻辑最清晰，也不会偷偷猜测用户想装哪些题。

### 装题数量规则

- 作者需要先明确填写“装题数量”
- 系统会按当前标签和筛选条件去匹配题目
- 只要匹配题数量少于目标数量，就直接失败并提示，不会偷偷只装一部分
- 只有数量足够时，才会精确装入指定数量的题目

### 行为语义

“按当前标签装入指定数量”当前是：

- 用匹配结果直接覆盖当前已选题目列表
- 覆盖后的题目数量与填写数量一致
- 之后仍然可以继续手动删除、补充或调整顺序

所以它更像：

- 先明确装入一套固定数量的题
- 再手动修一修

## 为什么不直接后台自动抽题

当前项目依然坚持：

- fail fast
- 不做过多猜测
- 不搞隐式补题

所以现在不会做：

- 自动猜题量
- 数量不够时偷偷只装一部分
- 自动补齐缺口
- 自动帮你删掉不想要的题

这些都交给作者工作台里的显式操作来完成。

## 语义日志增强

之前 Loki 里虽然已经能看到日志，但很多内容还是：

- `request_started`
- `request_finished`

这类请求级日志。

这轮补的重点，是把更多“业务动作”打成明确事件。

### 新增或强化的日志类型

#### 知识库

- `knowledge_documents_listed`
- `knowledge_document_viewed`
- `knowledge_document_imported`
- `knowledge_document_updated`
- `knowledge_document_archived`

#### 题库

- `question_drafts_listed`
- `question_tags_listed`
- `question_viewed`
- `question_preview_generated`
- `question_drafts_saved`
- `question_updated`
- `question_access_denied`

#### 考试

- `exam_papers_listed`
- `exam_paper_viewed`
- `exam_paper_created`
- `exam_paper_updated`
- `exam_assignment_created`
- `exam_attempt_started`
- `exam_attempt_viewed`
- `exam_attempt_graded`
- `exam_paper_access_denied`
- `exam_paper_management_denied`
- `exam_attempt_access_denied`

#### 权限

- `authoring_permission_denied`

### 当前日志风格

当前日志里会优先带这些上下文字段：

- `request_id`
- `actor_email`
- `resource_id`
- `error_code`
- `tag_names`
- `question_count`
- `result_count`

这让 Grafana 里按业务问题排查会更顺，而不是只看到一堆服务器启动信息。

## 系统日志噪音控制

为了让 Grafana 更聚焦 TalentHub 本身，这次还把：

- `uvicorn`
- `uvicorn.error`

的日志级别压到了 `WARNING`。

这样默认不会再把：

- 启动完成
- 监听端口
- 进程关闭

这类系统噪音放在最前面。

## 怎么看效果

### 在考试页

你现在可以：

1. 选标签
2. 填“装题数量”
3. 点“按当前标签装入指定数量”
4. 再手动删掉不想要的题
5. 或继续手动补题
6. 点“刷新候选题”重新拉一遍筛题结果

### 在 Grafana

建议直接查：

```logql
{job="talenthub-backend"} | json | event=~"knowledge_.*|question_.*|exam_.*|authoring_permission_denied"
```

只看考试：

```logql
{job="talenthub-backend"} | json | event=~"exam_.*"
```

只看访问拒绝：

```logql
{job="talenthub-backend"} | json | event=~".*_denied"
```

这样会比只盯着 request 日志清楚很多。
