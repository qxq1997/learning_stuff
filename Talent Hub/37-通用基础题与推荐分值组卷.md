# 通用基础题与推荐分值组卷

## 这次改了什么

这轮新增了两条直接影响出题和组卷的能力：

1. 支持不依赖知识库文档的通用基础题生成
2. 支持 AI 为每道题给出推荐分值，并在考试编纂页按推荐分值装配

## 通用基础题

之前系统生成题目主要依赖三类来源：

- 知识库文档
- 粘贴文本
- 网页 URL

现在新增第四类来源：

- 通用基础题

这个入口适合：

- Python 基础
- 数据结构
- 算法基础
- SQL 基础
- Linux 基础

这类题不要求你先上传文档，也不要求提供企业内部上下文。

### 使用方式

在出题工作台中选择“通用基础题”标签页，填写：

- 主题
- 补充要求（可选）
- 题型
- 难度
- 数量
- 标签

例如：

- 主题：`Python 基础`
- 补充要求：`只考列表、字典、函数和异常处理`

然后系统会直接调用模型生成题目预览。

### 后端语义

这类题会以 `source_kind = common_topic` 保存。

它和文档型来源的差别是：

- 不要求题目“引用企业文档”
- 不要求 `source_excerpt` 是文档原文摘录
- `source_excerpt` 在这里表示“该题考察的知识点依据”

也就是说，这里的 `source_excerpt` 更像：

- `列表推导式适合从可迭代对象快速生成新列表`
- `字典的 get 方法可避免键不存在时直接抛错`

而不是网页或文档中的逐字摘录。

## 推荐分值

现在模型在生成每道题时，除了返回：

- 题干
- 选项
- 标准答案
- 解析
- 来源摘录

还会额外返回：

- `recommended_score`
- `recommended_score_reason`

### 设计原则

这里遵守的不是“AI 直接决定考试最终分值”，而是：

- AI 给推荐分值
- 作者可手动调整
- 试卷最终使用的还是显式保存的每题分值

这样做的好处是：

- 题目难度和分值更一致
- 代码题和复杂题天然可以更高分
- 又不会把考试正确性完全交给模型自由发挥

## 考试编纂页怎么用

考试编纂页现在支持三套分值操作：

1. 先按标签和数量装题
2. 一键按推荐分值填充当前已选题
3. 一键把当前已选题统一改成默认每题分值

同时每道已选题都支持单独改分。

### 当前规则

- 自动按标签装题时，默认优先使用题目的推荐分值
- 如果某道题没有推荐分值，就回退到“默认每题分值”
- 创建试卷时会把每道题自己的 `score` 单独提交到后端
- 总分和及格分按当前选中题目的真实分值实时计算

## 为什么这样做

这套设计符合当前项目原则：

- fail fast
- 不做隐式猜测
- 不偷偷补题
- 不偷偷改分

也就是说：

- 题量由作者显式决定
- 分值可以由 AI 推荐，但最终是显式可见、可改、可保存的

## 当前边界

当前已经支持：

- 通用基础题生成
- 推荐分值展示
- 推荐分值保存到题库
- 组卷时按推荐分值装配
- 手动覆盖每题分值

当前还没有做：

- 基于历史答题表现动态调整推荐分值
- 根据岗位自动生成整套分值策略
- 按考试目标总分自动规划每题分值分布

这些后面可以在现有基础上继续叠加，但现在第一版已经把核心链路打通了。
