## 阅读策略

- 想先理解全局：从 `40` 开始。
- 想先看登录、部门、权限：先读 `41`。
- 想先看 AI 出题、知识库、网页抓取：先读 `42`。
- 想先看题库、组卷、考试、判卷：先读 `43`。
- 想先看日志、缓存、错误、配置、可观测性：先读 `44`。
- 想快速看项目价值、当前边界和下一阶段路线：先读 `45`。
- 想理解为什么作者工作台和成员考试入口要彻底分开：先读 `46`。
- 想看管理员如何观察团队考试表现、趋势变化和擅长领域：先读 `47`。
- 想看为什么现在日志会更偏业务步骤、而不是 API 噪声：先读 `48`。
- 想看整个系统为什么要“业务日志优先、错误信息直说人话”：先读 `49`。
- 想看为什么题目和考试现在是永久删除，以及删除会如何级联清理依赖：先读 `50`。
- 想看知识库如何真正进入“检索 -> 出题 -> 判卷 -> 学习建议”闭环：先读 `51`。
- 想直接理解为什么要把 RAG 检索结果显式暴露给用户看：先读 `52`。
- 想看为什么要做跨知识库检索，以及出处引用为什么要统一表达：先读 `53`。
- 想看为什么现在可以“不先选文档，直接基于整个知识库出题”：先读 `54`。
- 想看 TalentHub 后续为什么要从轻量 embedding 升级到父子 chunk、BGE-M3、Qdrant 和 reranker：先读 `55`。
- 想看这一轮为什么先只把 `BGE-M3` 接成 dense provider，而不是一次性把整个 hybrid 栈都切掉：先读 `56`。
- 想看为什么现在已经把 Qdrant 接进 dense retrieval 主路径，以及它和 PostgreSQL 的边界怎么切：先读 `57`。
- 想看为什么默认 embedding 切回 OpenAI，同时还保留 BGE 的切换能力：先读 `58`。
- 想看为什么默认 embedding 现在改成 `OpenAI text-embedding-3-large`，同时仍保留 `BGE` 的切换能力：先读 `58`。
- 想看 `bge-reranker-v2-m3` 和 `Ragas + ranx` 怎么进入主链路：先读 `59`。
- 想看 `BGE-M3` 的 sparse / multi-vector 作为高级 hybrid 路线怎么接进 `Qdrant`：先读 `60`。
- 想系统梳理当前真实 schema 到底有哪些表、怎么分组、三条主线怎么串：先读 `61`。
- 想细看身份、知识库、RAG、题库这些表之间的关系：先读 `62`。
- 想细看试卷、分配、作答、判卷、学习和分析表之间的关系：先读 `63`。
- 想理解为什么有些关系用强外键、有些不用，以及数据库后续还能怎么升级：先读 `64`。
- 想看为什么当前推荐把 reranker 切到线上 Voyage，同时保留本地 BGE fallback：先读 `65`。
- 想看成员侧为什么新增“我的学习”，以及专项练习如何和现有考试系统复用闭环：先读 `66`。
- 想看为什么出题和判卷不再继续阻塞请求链路，以及当前 Worker/任务表如何承载这些重操作：先读 `67`。
- 想看 TalentHub 如何把“系统自行判断”收敛成可解释的检索计划和学习建议计划：先读 `68`。
- 想看第一版 decision plan 之后，为什么还要继续抽象出 DecisionContext / PolicyEngine，并把专项练习历史接成 practice memory：先读 `69`。
- 想看为什么学习建议不能一遍遍推相同资料，以及专项练习效果怎么真正回流到 mastery 画像：先读 `70`。
- 想看 memory 是否按用户隔离、为什么要分层、k 怎么定、什么时候扩召、top-k 分数怎么解读：先读 `71`。
- 想看这个项目一路实现过程中到底踩了什么坑、为什么改方案、问题是怎么解决的：先读 `72`。
- 想系统看清楚 TalentHub 每一项核心技术选型到底解决了什么问题、牺牲了什么、为什么不是别的方案：先读 `73`。
- 想知道现在系统检索能力到底到了哪一步、现有 benchmark 强弱项是什么、dashboard 应该怎么看：先读 `74`。
- 想直接看这轮真实 benchmark 的运行配置、场景结果、弱项案例和当前检索能力结论：先读 `75`。
- 想看完全基于当前真实知识库文档、而不是 synthetic 文档的 dense-only 与 reranker 对照结果：先读 `76`。

## 与历史文档的关系

`00-38` 号文档仍然保留，主要承担两个作用：
