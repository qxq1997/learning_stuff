# 模块亮点、已解问题与未来演进

## 1. Identity 模块

### 当前亮点

- 使用正式 Bearer token 完成登录与注册
- 权限边界围绕“动作”而不是“页面”
- `ActorContext` 让服务端身份语义统一

### 已解决问题

- 移除了开发期手动切邮箱的做法
- 修复匿名写知识库的问题
- 统一了部门管理员和普通成员的访问边界

### 下一步

- 用户管理页
- 更细粒度角色
- refresh token / 会话管理

## 2. Knowledge 模块

### 当前亮点

- 同时支持文本导入和网页 URL 导入
- 知识文档支持预览、编辑、归档
- 与动态网页抓取能力自然衔接
- 已接入切片、向量索引与检索
- 已能为出题和考后学习建议提供依据片段

### 已解决问题

- 企业知识来源不再只能靠本地文件
- 知识文档不再是一次性导入，具备可维护性
- 知识库终于不只是“存文本”，而是能真正参与题目生成和反馈闭环

### 下一步

- 文档版本历史
- 归档恢复
- 检索质量评测
- 文档级与部门级检索范围控制
- 将轻量检索底座升级为父子 chunk + BGE-M3 + Qdrant + bge-reranker-v2-m3

## 3. Question Bank 模块

### 当前亮点

- 题目生成 preview 与正式入库分离
- 标签、搜索、分页、编辑、详情已成体系
- 支持单选、多选、代码题
- 支持通用基础题，无需上传文档
- 知识库出题已改为基于检索片段，而不是整篇文档直塞模型
- 保存题目时会持久化题目与知识片段的引用关系

### 已解决问题

- 题库不再是 AI 一次性输出结果
- 题目可以版本化演进
- 题目可复用到不同考试

### 下一步

- 更强的题目质量评估
- 代码题执行沙箱
- 标签推荐与标签治理

## 4. Exam 模块

### 当前亮点

- 按标签和数量半自动装题
- 试卷支持公私有边界
- 分配、开始、提交、判卷链路完整
- 作答结果有逐题反馈
- 个人考试中心和作者工作台入口已拆开
- 判卷后会按题回查知识片段，给出推荐回看资料

### 已解决问题

- 考试不再只是静态试卷对象
- 私有考试和公共考试的访问模型已明确
- 试卷被使用后不可编辑，保护历史一致性
- 普通成员不再需要进入作者页面寻找考试入口

### 下一步

- 答题自动保存
- 考试趋势与能力画像联动
- 更细的考试历史筛选和导出

## 5. Analytics 模块

### 当前亮点

- 已有管理员指标页
- 指标按部门或平台范围聚合
- 每个人都有成绩趋势与标签强弱项
- 指标日志具备明确业务语义

### 已解决问题

- 管理员终于能看到团队考试表现，而不是只看到系统对象数量
- 标签从筛题工具扩展成可分析的能力线索
- 首页和指标页的角色语义更一致

### 下一步

- 时间区间筛选
- 部门对比和岗位对比
- metric snapshot 正式接管高频指标

## 6. Shared 模块

### 当前亮点

- 结构化日志
- 统一错误模型
- 缓存策略显式化
- AI provider 工厂化

### 已解决问题

- 日志从系统噪音转向业务事件
- 配置项集中管理
- 外部能力接入不再散落到业务逻辑中

### 下一步

- tracing
- smoke 回归自动化
- 更完整的本地可观测性面板

## 7. 当前最值得肯定的设计成果

- 系统已经从“功能叠加”开始转向“语义清晰的模块化单体”
- 路由层开始变薄，领域和应用边界更容易读懂
- 中文文档开始从“开发记录”转向“主线设计说明”

## 8. 当前仍然需要警惕的风险

- 部分 application use case 仍可继续抽薄
- 某些前端页面仍承担过多工作台职责
- 当前 RAG 已闭环，但检索底座仍处于轻量实现阶段
- 代码题当前仍主要依赖 AI 判分，尚未引入执行级验证
- 图片知识仍未形成正式多模态检索通道

## 9. 建议的下一阶段路线

1. 继续收敛接口层与依赖装配
2. 评测并升级检索栈：父子 chunk、BGE-M3、Qdrant、bge-reranker-v2-m3
3. 做学习结果沉淀与能力画像
4. 为代码题接入执行沙箱
5. 建立稳定自动化回归与评测体系
