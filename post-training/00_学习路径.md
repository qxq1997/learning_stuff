# 后训练(Post-Training)学习路径(2026 工程落地版)

## 这个专题给谁看

- 你是 3-8 年后端/平台工程师,想系统补齐"从开源模型到线上能用"这段工程栈。
- 你清楚预训练对普通团队门槛太高(动辄几千万美金的 GPU Cluster、万卡级集群调度、PB 级数据处理),但你发现 **后训练 + 推理部署** 这一侧已经被极大地平民化了:一张 4090 能跑 QLoRA,一张 H20 能跑 7B-32B 的在线推理,甚至 Mac M 系列都能拉起一个 30B 级别的服务。
- 你不满足于"Ollama pull 一下跑起来"这种 Demo 级理解,想能在面试或技术评审里讲清楚:`LoRA 的秩到底怎么选`、`GRPO 比 PPO 好在哪`、`vLLM 的 PagedAttention 解决的是什么物理问题`、`AWQ 和 GPTQ 的数学原理差异`、`为什么 FP8 训练 2025 才真正普及`、`Reward Hacking 具体长什么样`、`Agentic RL 和普通 RL 到底差在哪里`。
- 你希望这套知识能落到`动手`:用自己的数据微调一个 7B 模型、用 vLLM 部起来、做成线上服务、还能评估它有没有变笨。

## 为什么专门开这么一个专题

市面上的"大模型课"绝大多数有三个问题:

1. **偏预训练**:讲 Transformer、讲 Scaling Law、讲 MoE,但预训练普通人根本做不了,学了只能当饭后谈资。
2. **偏应用层**:讲 Prompt、讲 RAG、讲 Agent,但这一层已经被 RAG / Agent 专题覆盖(见同级目录下的 `RAG/` 和 `AI Agent/`)。
3. **中间那段断层**:怎么把一个开源 checkpoint 训得更贴合你的业务、怎么压到单卡能跑、怎么在 200 QPS 下保持 P99 延迟、怎么在保留通用能力的同时加上领域知识——这一段正是普通公司最能吃到红利的地方,但几乎没有体系化课程。

这个专题就补这一段。目标是:**让一个后端工程师能独立完成"下载开源模型 → 本地跑通 → 量化压缩 → SFT/LoRA → 对齐训练 → Agent 能力训练 → 推理部署 → 在线评估"的全链路**,并且每一步都能讲出为什么这么做、代价是什么、换一条路径会怎样。

## 写作原则

- **深度优先**:每一章都要追到原理层。讲 LoRA 必须讲低秩假设和 ΔW 的结构;讲 PagedAttention 必须讲操作系统的分页启发和 KV Cache 的物理碎片;讲 GRPO 必须讲它相对 PPO 去掉了什么;讲 DPO 必须从 PPO 的闭式解推导出它为什么能跳过 Reward Model。
- **有时代感**:每一章都会标注`2022 → 2023 → 2024 → 2025 → 2026` 的演化路径,让你知道每个技术为什么在当年出现、被谁替代、现在还值不值得学。
- **重工程权衡**:同样一个目标(比如"让 7B 模型学会 SQL 生成"),SFT / LoRA / DPO / RLAIF / Distillation / Agent RL 都能做,区别在`数据量、GPU 时数、是否损伤通用能力、是否可回滚、是否需要 reward`。每章都会给出多维对比。
- **代码可跑**:示例全部基于 2026 主流栈——transformers、trl、peft、unsloth、LLaMA-Factory、axolotl、veRL、vLLM、SGLang、llama.cpp。版本号都会标出来。
- **不堆名词**:每个新技术都要回答`它替代了谁、改进了什么指标、代价是什么、什么时候不用`。

## 每章的标准结构

- **学习目标**:读完你能独立做什么(能回答什么问题、能跑起什么代码、能设计什么架构)。
- **问题背景**:这项技术为什么被发明,它在解决什么具体痛点。
- **演化时间线**:`2022 → 2023 → 2024 → 2025 → 2026` 每年发生了什么。
- **核心原理**:讲透算法/数学/系统设计,该写公式写公式,该画图画图(ASCII 或 Mermaid)。
- **代码示例**:Python,基于 2026 主流库版本,能跑。
- **工程权衡**:成本、显存、延迟、稳定性、可维护性怎么平衡。
- **面试 / 评审怎么问**:真实深挖问题 + 回答框架。
- **小结 + 自测题**:3-5 条 takeaway + 3 个自测问题。
