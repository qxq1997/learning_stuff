# 后训练 - 第 1 课：后训练全景图——为什么预训练交给大厂,后训练才是普通人的战场

## 学习目标(本节结束后你能做到什么)

- 能画出一张"大模型从无到有再到上线"的四段式生命周期图,说清楚每一段在干什么、需要什么资源、由谁来做。
- 能明确告诉同事 / 面试官:`预训练 vs 中期训练 vs 后训练 vs 推理优化` 各自的输入输出是什么、算力门槛差几个数量级。
- 能回答这个灵魂问题:"我们公司想接大模型,到底该从哪一段切入?",并给出一套基于算力、数据、合规、团队能力的决策框架。
- 能列出后训练这一层 2022-2026 的主干技术路线(SFT → LoRA → RLHF → DPO → GRPO → 推理加速 → Agentic RL),知道每个技术是在解什么上一代留下来的问题。
- 知道 2026 年常见的工具链全家桶(HuggingFace Transformers、PEFT、TRL、Unsloth、LLaMA-Factory、Axolotl、veRL、vLLM、SGLang、llama.cpp)各自的角色和边界。

## 一、先划清楚一件事:你以为的"训练大模型",其实是四件事

在继续往下讲之前,我们必须先统一语言。很多人一提到"训练大模型"就在脑子里闪过一个笼统的画面——大量 GPU、大量数据、几个月——然后默认这是一件遥不可及的事。这种认知的最大问题是把一条流水线上的四个截然不同的阶段压扁了。你看不清阶段,就看不清成本结构,也就看不清`普通团队能在哪一段切入`。

一个现代大模型从"什么都没有"到"线上能用",严格来讲要经过 **四个阶段**:

```
[ 原始文本/代码/网页 ]
        │
        ▼  ───────── 阶段 1:预训练(Pre-training)
[ Base Model ]             输入:数万亿 token 的无监督语料
        │                  输出:会"续写"但不会"听话"的底模
        │                  资源:万卡级 GPU,数千万-数亿美金
        │                  谁在做:OpenAI / Anthropic / Google / Meta / DeepSeek / Qwen / Mistral / 少数国家队
        │
        ▼  ───────── 阶段 2:中期训练(Mid-training / Continued Pre-training)
[ Mid-trained Base ]       输入:数百亿-千亿 token 的高质量/领域语料(代码、数学、推理轨迹、长上下文)
        │                  输出:更擅长某类能力的底模(code base、math base、long-context base)
        │                  资源:千卡级 GPU,数百万-数千万美金
        │                  谁在做:大厂 + 少数财力充裕的中厂,开源社区几乎没有
        │
        ▼  ───────── 阶段 3:后训练(Post-training)
[ Aligned Model ]          输入:数千-数百万条指令 / 偏好 / 可验证任务数据
        │                  输出:能听指令、能对齐偏好、能解推理题、能用工具的 Instruct Model
        │                  资源:8-1024 卡,几千-几十万美金,小规模 4090 单卡都能开工
        │                  谁在做:几乎所有人——大厂、中厂、创业团队、个人开发者
        │
        ▼  ───────── 阶段 4:推理优化与部署(Inference / Serving)
[ 在线服务 ]               输入:训练好的权重 + 业务流量
                           输出:稳定、低延迟、高吞吐的在线推理服务
                           资源:从单卡消费级到多卡 H100,弹性极大
                           谁在做:所有要把大模型用到产品里的团队
```

这四段的`算力、数据、团队能力要求`是完全不同的数量级,混在一起讲会让人彻底丧失判断力。我们来一段一段拆开。

### 1.1 预训练(Pre-training):大厂和国家队的战场

预训练干的事情非常朴素:给模型看海量文本,让它学会`在看到前 n 个 token 的前提下,预测第 n+1 个 token 的概率分布`。就这一件事,重复几万亿次。

它之所以贵,是因为这几万亿次要在几万亿 token 规模的语料上、几百亿参数规模的模型上、连续几个月不停机地跑,中间任何一台机器挂了都要有 checkpoint 能继续。成本结构大概是:

- **GPU 时长**:Llama 3.1 405B 用了 3.84×10^25 FLOPs 的计算量,按 H100 的 2 PFLOPS(BF16)算,大约是 1.6 亿 GPU-小时。按 H100 云服务每小时 2 美元算,纯算力成本就是 3.2 亿美元。Meta 实际报告的是 16k 张 H100 训了 54 天。
- **数据处理**:15T token 的高质量语料,光是去重、过滤低质、去毒、分类,就需要专门的数据 pipeline 团队。
- **工程团队**:分布式训练的坑(loss spike、通信带宽、checkpoint 容错、NaN 排查)需要一个 20-50 人的 Infra + Research 团队磨合一年以上。
- **合规与数据采购**:语料版权、审核、过滤黑名单,这些是隐性成本,但在一些地区可能比算力还贵。

结论非常清楚:**预训练不是一个可以"练一练"就能切入的赛道**。即使你有一亿美金,没有稳定的高质量语料、没有懂分布式的 Infra 团队,你最大概率是烧钱复刻一个比 Qwen3 / DeepSeek-V3 / Llama 3 差的模型,还赶不上开源节奏。这就是为什么 2024 年之后,国内只剩下几家真正在做基座,其他都退回到`在开源基座上做后训练`这条路。

### 1.2 中期训练(Mid-training):2024 年之后才被独立命名的阶段

2023 年之前,大家把这一段隐藏在预训练或后训练里。2024 年之后,社区开始把它独立出来,叫 "mid-training" 或 "continued pre-training"。它的典型场景是:

- **领域继续预训练**:拿 Llama 3 70B 作为起点,灌 5000 亿 token 的医疗文献,得到一个医疗底模。
- **能力注入**:给底模灌高质量代码(得到 code base)、数学证明(得到 math base)、长上下文文档(得到 long-context base)。DeepSeek-Coder-V2、Qwen2.5-Math、Yi-9B-200k 这类模型都在这一段做文章。
- **推理轨迹混入**:在 2025 年之后的 `thinking model` 训练里,mid-training 阶段会把大量带 CoT(chain-of-thought) 的数据混进去,让底模天然具备展开推理的倾向。

它的资源仍然在百万到千万美金级别,多数中小团队还是做不起,但已经有一些财力较厚的垂直公司(医疗、法律、金融)开始做自己的 mid-training。2026 年的趋势是:**mid-training 的门槛会继续下降,因为开源基座越来越好,只需要`打补丁`而不是`从零`**。

### 1.3 后训练(Post-training):本课程的主战场

终于轮到我们要啃的这一段了。后训练的定义是:**拿一个预训练(或中期训练)过的 Base Model,用各种有监督 / 偏好 / 强化学习手段,把它调教成一个能用、好用、符合偏好、符合安全要求的产品模型。** 它是从 `Base → Instruct` 的那一步飞跃。

为什么这一段是普通人的战场?几个硬指标对比:

| 维度 | 预训练 | 后训练(7B 模型 LoRA SFT) | 后训练(7B 模型全参 SFT) |
| --- | --- | --- | --- |
| 数据量 | 10^13 token | 10^3 - 10^6 条样本 | 10^5 - 10^7 条样本 |
| 单次训练时长 | 数周-数月 | 几小时-1 天 | 1-7 天 |
| GPU 数量 | 几千-几万张 | 1 张 4090 就能开跑 | 8 张 A100 起 |
| 算力成本 | 数百万-数亿美金 | 几十元-几千元(云租) | 几千-几万元 |
| 工程复杂度 | 分布式、容错、数据 pipeline | Python + HuggingFace + 一个配置文件 | 加上 DeepSpeed / FSDP |
| 失败恢复成本 | 极高(重训一遍要几百万) | 极低(重来一次几小时) |

**一张 4090 + 一台云 GPU + 一个周末**,就能完成一个 7B-14B 模型在你业务数据上的 LoRA 微调,这是 2024 年之后的现实。这个"平民化"的根源是三件事叠加:

1. **开源底模质量爆表**:Llama 3、Qwen3、DeepSeek-V3、Mistral 等都已经是"接近 GPT-4 基础能力"的水平,你在这种底座上做后训练,起点已经非常高。
2. **参数高效微调(PEFT)成熟**:LoRA / QLoRA / DoRA 让你只训练 1% 的参数就能达到 90% 以上的全参微调效果,显存需求从几百 G 降到十几 G。
3. **工具链工业化**:HuggingFace 全家桶、Unsloth、LLaMA-Factory、Axolotl 把底层细节封装到"改一个 yaml 就能训"的程度,工程门槛从"博士后"降到"会写 Python 就行"。

后训练这一段的**内部结构**,也远比外人想象的复杂。它不是"微调"一件事,而是至少 8 个子方向:

```
[ Base Model ]
        │
        ▼
┌──────────── 后训练子系统地图 ────────────┐
│                                          │
│  1. SFT(Supervised Fine-tuning)        │
│     → 教模型"听指令"的基本功            │
│                                          │
│  2. PEFT(LoRA、QLoRA、DoRA、…)         │
│     → 让 SFT 的算力门槛降到单卡级       │
│                                          │
│  3. Preference Alignment                 │
│     → RLHF / DPO / IPO / KTO / ORPO      │
│     → 让模型知道"好答案 vs 更好答案"    │
│                                          │
│  4. Reasoning RL                         │
│     → GRPO / DAPO / GSPO                 │
│     → 让模型真的会"解题",不是背答案    │
│                                          │
│  5. Distillation                         │
│     → 从 671B 大模型蒸出 7B / 32B       │
│                                          │
│  6. Tool-use / Agent 训练                │
│     → Function Calling SFT、Agent RL     │
│                                          │
│  7. Long-context 后训练                  │
│     → YaRN / RoPE 扩展、长文档 SFT       │
│                                          │
│  8. 量化与推理优化                       │
│     → GPTQ / AWQ / GGUF / FP8 / 投机解码 │
│                                          │
└──────────────────────────────────────────┘
        │
        ▼
[ Instruct / Chat / Reasoning / Agent Model ]
```

本课程的第 09-21 章会把上面每一条线讲透,第 22 章会把它们串起来做一个完整项目。

### 1.4 推理优化与部署:常被低估,但最能省钱的一段

很多团队愿意在微调上花几十万,但在推理上用一句 `transformers.generate()` 就上线了——这是 2024-2026 最常见的资源浪费。一个没经过推理优化的 7B 模型,在 A100 上大约能做到 30-50 tokens/s 的吞吐;用上 vLLM + PagedAttention + Continuous Batching,这个数字能飙到 2000-5000 tokens/s(batched)。**算力成本直接除以 50-100**,这就是为什么字节、阿里、Meta 都把推理引擎当成核心基础设施。

推理优化这一段的子方向,我们第二部分会详细展开:

- **KV Cache 管理**:PagedAttention、Prefix Caching、KV Cache Offloading。
- **批处理**:Continuous Batching、Chunked Prefill、Prefill/Decode 分离。
- **Kernel 优化**:FlashAttention-1/2/3、Triton Kernel、Machete、Marlin。
- **量化推理**:INT4/INT8/FP8 推理,量化感知 kernel。
- **投机解码**:Medusa、EAGLE-3、Lookahead、Multi-Token Prediction。

## 二、2022-2026 后训练技术演化时间线

把事情放进时间轴里看,才能理解`为什么会这样发展`。下面是一条浓缩的主干线,尽量避免堆砌名词,只点出每年最大的范式跳变。

**2022:RLHF 元年**
- InstructGPT(2022.03)首次证明:`SFT → RM → PPO` 这套三段式能把 GPT-3 调成能对话的 ChatGPT。
- 但这套 pipeline 重到什么程度?需要同时托管 4 个模型(SFT、RM、ref、actor),PPO 还极其难调。普通团队根本做不了。

**2023:LoRA 和开源基座的爆发**
- Llama 1/2 开源(2023.02/07),第一次让"拿一个顶级底模做微调"变成普通人能做的事。
- LoRA(2021 提出,2023 全面普及)让 7B-13B 模型 SFT 的显存需求从 200G 降到 20G。
- QLoRA(2023.05)进一步把它压到一张 24G 4090 能跑 33B 的程度。
- Alpaca、Vicuna、Dolly 一批`开源指令数据集`出现,SFT 的数据问题也开始有解。

**2024:DPO 时代 + 工具链工业化**
- DPO(2023.05 提出,2024 年全面铺开)把 RLHF 的四模型 pipeline 压缩到两模型,公式漂亮、训练稳定,几乎所有开源对齐模型(Llama 3、Mistral、Zephyr)都转向 DPO 系。
- Unsloth、LLaMA-Factory、Axolotl 三大训练框架崛起,把 SFT + LoRA + DPO 封装成"改一个 yaml"。
- vLLM(2023 发布,2024 爆发)用 PagedAttention 重写推理,把开源推理引擎拉到工业级。
- 蒸馏成为开源主流方法——OpenHermes、Nous、Dolphin 系列都是基于大模型蒸馏 + SFT。

**2025:Reasoning RL 元年**
- DeepSeek-R1(2025.01)发布,证明`仅靠 RL + rule-based reward`就能让模型涌现出长 CoT、反思、回溯等能力,不需要 SFT 冷启动。这是 2022 以来最大的范式跳变。
- GRPO(Group Relative Policy Optimization)取代 PPO,去掉 critic model,对齐训练的显存和工程复杂度再降一个台阶。
- Qwen3、Kimi-K2、DeepSeek-V3、Mistral Large 2 等一批"思考模型"出现,模型标配 `think` 模式。
- 长上下文训练从 YaRN / NTK 时代进入原生稀疏注意力时代(DeepSeek NSA、Kimi MoBA)。
- Agent / Tool-use 训练第一次有了像样的开源栈(veRL、OpenAgent、SkyRL)。

**2026:Agentic RL + 推理服务分层**
- RL 训练不再只针对单步问题(数学题、代码题),开始面向多步 Agent 任务(SWE-Bench、WebShop、OSWorld)——这被称为 "Agentic RL" 或 "RL for Agents"。
- 推理服务架构走向 Prefill / Decode 分离(Disaggregated Serving),Mooncake、DistServe 思路成为主流引擎(vLLM / SGLang)的原生能力。
- FP8 训练全面铺开(H100 / H200 / B200 硬件就绪),训练成本进一步降低。
- 开源社区的 "从 API 到自托管" 迁移完成:一个中等规模公司可以完全不依赖闭源 API,全链路自主后训练 + 自主服务。

**主线概括**:后训练这一层,从 2022 到 2026,走的是一条`门槛持续降低、能力持续提升、工程链条持续规范化`的路线。**这也是为什么我们要学这一层——因为它每一年都在给你开新口子**。

## 四、一个灵魂决策:你的团队该投哪一段?

如果你是带一个 5-20 人小团队的技术负责人,面对大模型落地的选择,可以按下面这张表做决策。这张表的核心是:**不要越级投资**。越级投资的典型表现是没有 10 亿美金预算却想复刻一个基座,或者没有一张 4090 就想搞 Agentic RL。

| 你现在的资源 | 最合适的投资段 | 不该碰的段 |
| --- | --- | --- |
| 没有 GPU,没有训练工程师 | 调用 API + Prompt + RAG + Agent | 微调、预训练 |
| 1-2 张 4090 / 4090 级别云卡 | SFT + LoRA / QLoRA,推理走 llama.cpp / Ollama | 全参微调、RLHF |
| 8 张 A100/H100,2-5 人团队 | 全参 SFT、DPO、GRPO、量化部署 | 从零预训练 |
| 32-128 张 H100,5-20 人团队 | 中期训练、大规模 RL、自研蒸馏 | 从零基座预训练(除非有独占数据) |
| 1000+ 卡,50+ 人 Infra | 全栈预训练 + 后训练 | - |

90% 的读者会落在前三档。本课程的正文(第 02-22 章)就是针对前三档团队写的——默认你有 1-8 张 GPU、想做出线上能用的东西、不想把钱烧在基座上。

## 五、小结

1. **"训练大模型"不是一件事,是四件事**:预训练、中期训练、后训练、推理部署。四件事的算力门槛差几个数量级,普通团队只应该碰后两件。
2. **后训练是 2024-2026 最大的工程红利区**:开源底模越做越好、PEFT 技术成熟、工具链工业化,让单卡级别的后训练成为现实。
3. **后训练内部至少有 8 个子方向**:SFT、PEFT、Preference Alignment、Reasoning RL、Distillation、Tool-use、Long-context、Quantization & Serving。本课程每条线都会啃。
4. **演化主线**:RLHF(2022) → LoRA + 开源(2023) → DPO + 工具链(2024) → Reasoning RL(2025) → Agentic RL + Disaggregated Serving(2026),门槛逐年下降,能力逐年上升。
5. **决策框架**:按你能掌握的 GPU 数量反推该投哪一段。永远不要越级投资。
