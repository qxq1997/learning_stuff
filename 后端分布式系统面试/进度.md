## 重点关注的理解难点

- 容易把“分布式系统”理解成“微服务 + Redis + MQ”的组件拼盘
- 容易只会讲正向流程，不会讲失败路径、补偿路径和恢复路径
- 容易把缓存问题和一致性问题割裂来看
- 容易高估分布式锁的作用，低估状态机、幂等键、对账机制的重要性
- 容易只说技术名词，讲不清为什么这么选、代价是什么、替代方案是什么

## 评估标准（用于自适应推进）

- 完全掌握：答案准确，能举一反三 -> 进入下一课
- 基本掌握：答案大致正确，有小偏差 -> 简短纠正后进入下一课
- 部分理解：概念混淆或有明显误区 -> 生成补充课（如 `01b_补充.md`）
- 尚未理解：关键问题回答错误或不知道 -> 回到更基础层重新搭建理解
