# 后端面试导向的算法题学习路径

## 学习目标

这条学习路径的目标，不是把你训练成刷题机器，而是帮你建立一套更适合后端面试的算法认知框架：

- 你能把常见题目迅速归类到几个经典套路，而不是每题都从零开始想。
- 你能说清楚每个套路解决的核心矛盾是什么，什么时候该用，什么时候不该用。
- 你能把算法题和真实后端场景连接起来，例如去重、频次统计、Top K、区间合并、树形结构遍历、缓存淘汰等。
- 你能写出简洁可解释的 Java 模板代码，但代码只是辅助，不会喧宾夺主。
- 你能在面试里先讲思路、再讲复杂度、最后给出核心实现，而不是一上来埋头写代码。

## 重要说明

这次内容会明确偏向“教程型文档”，也就是：

- 以文字总结、题型归纳、解题套路为主
- 以少量 Java 代码片段为辅助
- 不再把内容组织成一堆分散的代码文件

换句话说，这个主题的重点不是“给你一堆现成答案”，而是帮你建立一张算法题地图。  
真正适合后端工程师的学习顺序，应该是：

1. 先知道高频题型有哪些
2. 再知道它们各自的识别信号
3. 再知道通用模板怎么写
4. 最后才是按题目去练

## 知识图谱

这门主题建议按 5 个层次理解：

1. 题型识别层
   - 哈希表
   - 双指针
   - 滑动窗口
   - 前缀和
   - 二分查找

2. 数据结构层
   - 栈
   - 队列
   - 堆
   - 链表
   - 树

3. 搜索与递推层
   - BFS
   - DFS
   - 回溯
   - 动态规划

4. 面试表达层
   - 如何先讲暴力解
   - 如何识别瓶颈
   - 如何解释复杂度
   - 如何用 case 自测

5. 后端联系层
   - LRU
   - Top K
   - 区间合并
   - 树形结构遍历
   - 限流窗口和滑动窗口思维

## 推荐学习方式

- 先把第 1 课当成总纲，不要急着刷很多题。
- 读每个套路时，先记“适用信号”，再记“模板代码”。
- 不要在第一轮就追求题海，先把每个套路各挑 2 到 3 道经典题吃透。
- 每次做题都按同一个顺序复盘：题型、核心矛盾、数据结构、复杂度、边界条件。

## 学完后你应该能回答的问题

- 为什么很多算法题，本质上只是几个套路在不同外壳下的重复出现。
- 为什么后端面试更看重题型识别和复杂度分析，而不是冷门技巧。
- 为什么哈希、滑动窗口、堆、树遍历、二分这些题型特别高频。
- 为什么教程型模板比零散题解更适合建立长期可复用的能力。

## 代码约定

- 默认使用 `Java` 做示例，因为它最适合讲清楚数据结构选择。
- 代码会内嵌在文档里，只保留少量核心模板。
- 代码的作用是帮助你理解套路，不是把文档变成题解仓库。
