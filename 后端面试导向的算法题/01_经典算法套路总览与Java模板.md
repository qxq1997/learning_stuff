# 后端面试导向的算法题 - 第 1 课：经典算法套路总览与 Java 模板

## 学习目标（本节结束后你能做到什么）

1. 建立一张后端面试高频算法套路地图，知道常见题目大致落在哪几类。
2. 理解每个经典套路背后的核心思想，而不是只记题目和答案。
3. 掌握几段足够常用的 Java 模板代码，能把它们当成“思路锚点”。
4. 形成一套更适合面试的解题顺序：先识别题型，再讲思路，最后再写代码。

## 内容讲解（核心概念，用类比、例子、图示说清楚）

你这类需求其实很典型。很多后端工程师学算法题时，不是真的缺代码，而是缺一张“套路地图”。题刷得越多，反而越乱：这题像见过，但名字不同；那题代码似乎写过，但为什么这样写说不清。最后就会形成一种错觉，好像算法题只能靠题海硬记。其实不是。大多数面试高频题，本质上都可以归到少数几个经典套路里。真正有用的学习方式，不是收藏一堆题解，而是先把这些套路背后的信号、适用条件和 Java 写法搞清楚。

你可以把算法题理解成“用数据结构组织约束”。题目表面在讲数组、字符串、树、链表，但更深一层，它往往在问你：你是要快速查找，还是要维护一个动态区间，还是要保留局部最优，还是要层层扩展，还是要在有序空间里找边界。不同问题对应不同工具。面试官最想看的，不是你有没有见过原题，而是你能不能迅速判断“这道题的核心矛盾是什么”，再把它映射到合适的套路。

这节课我先不给你铺太多分支内容，而是先把最常见、最值得优先掌握的几类套路讲透。你可以把它们当成后端面试算法题的主干。

### 一、哈希表：把“查找”从遍历变成常数时间

哈希表几乎是后端算法题里最先该掌握的一类。它解决的核心问题是：当你在遍历过程中，需要快速判断一个元素是否出现过、某个值对应的位置是什么、某个频次累计到多少时，哈希表通常是第一选择。

这类题常见的信号包括：

- 是否存在
- 是否重复
- 两数之和
- 频次统计
- 去重
- 分组

你可以把它理解成一个“边走边记账”的过程。比如两数之和，暴力法是双重循环去找另一个数，这样是 `O(n^2)`。如果你在遍历时把已经看过的值记进 `HashMap`，就能把“找另一个数”的动作压成 `O(1)` 均摊时间。这个思路在很多题里都会反复出现。

下面是一段最典型的 Java 模板：

```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> indexByValue = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int need = target - nums[i];
        if (indexByValue.containsKey(need)) {
            return new int[] {indexByValue.get(need), i};
        }
        indexByValue.put(nums[i], i);
    }
    return new int[0];
}
```

这段代码重要的不是题目本身，而是模式：  
**一边遍历当前值，一边去哈希表里查“我需要的另一个值是否已经出现”。**

你学哈希表时，最该记住的是这个模式，而不是“两数之和”这个题名。

### 二、双指针与滑动窗口：处理连续区间问题

很多后端工程师会把双指针和滑动窗口混在一起，其实它们关系很近，但使用场景有区别。

双指针更像是一种控制遍历范围的方式，常见有两种：

- 左右夹逼：通常用于有序数组，比如两端往中间逼近
- 快慢指针：通常用于链表或原地数组处理，比如去重、找环、分区

滑动窗口可以看成双指针的一种“动态区间维护版”。它的核心不是两个指针本身，而是你维护的这个窗口 `[left, right]` 是否满足某个条件。

这类题的典型信号非常明显：

- 最长
- 最短
- 连续子数组
- 连续子串
- 满足某个条件的区间

比如“最长无重复子串”这题，很多人一开始会想到枚举所有子串再判断是否重复，但那样复杂度很差。真正应该抓住的是：这是一个连续区间，而且区间内有一个动态约束“不能重复”。这种题就应该优先往滑动窗口想。

下面是一段很经典的 Java 模板：

```java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> lastIndex = new HashMap<>();
    int left = 0;
    int answer = 0;

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (lastIndex.containsKey(c)) {
            left = Math.max(left, lastIndex.get(c) + 1);
        }
        lastIndex.put(c, right);
        answer = Math.max(answer, right - left + 1);
    }
    return answer;
}
```

这段代码的套路是：

1. `right` 负责扩张窗口
2. 当条件被破坏时，`left` 负责收缩或跳转
3. 每一步都维护一个当前最优答案

以后你遇到“连续区间 + 最长最短 + 条件动态变化”的题，脑子里就应该先冒出这个模型。

### 三、前缀和 + 哈希：处理“子数组和”“区间统计”这类题

前缀和单独拿出来并不难，但它一旦和哈希表结合，威力就会很大。这类题的本质通常是：你不是在找单个元素，而是在找某一段区间的累计结果是否满足条件。

常见信号包括：

- 子数组和等于 k
- 区间和
- 有多少个连续子数组满足条件
- 统计某种前后差值

很多人第一次做“和为 k 的子数组”时，会用双重循环枚举所有起点和终点。这样当然能做，但不够面试导向。更好的思路是：如果 `prefix[i] - prefix[j] = k`，那么问题就变成了在遍历到当前位置时，看看之前有没有出现过 `prefix[i] - k` 这个前缀和。这样又回到了“边走边查”的模型。

核心 Java 模板如下：

```java
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> countByPrefix = new HashMap<>();
    countByPrefix.put(0, 1);

    int prefix = 0;
    int answer = 0;
    for (int num : nums) {
        prefix += num;
        answer += countByPrefix.getOrDefault(prefix - k, 0);
        countByPrefix.put(prefix, countByPrefix.getOrDefault(prefix, 0) + 1);
    }
    return answer;
}
```

这类题要重点理解两件事：

- 前缀和负责把“区间问题”转成“两个前缀的差”
- 哈希表负责快速统计“某个前缀之前出现过几次”

这比单纯记模板更重要，因为你一旦理解这层，很多看起来不一样的统计题都会变成同一个东西。

### 四、堆：处理 Top K 和“动态最值”

堆，也就是优先队列，在后端面试里很常见，因为它天然适合处理“只关心前几名”或者“持续维护局部最优”的问题。

典型场景包括：

- Top K 高频元素
- 第 K 大元素
- 合并多个有序流
- 任务调度中的优先级选择

如果你每次都把全部元素排序，当然也可能能做出来，但未必是最贴切的解法。尤其当题目强调“实时维护”“只取前 K 个”“数据持续到来”时，堆的味道就非常浓。

下面这段代码体现的是“哈希统计频次 + 小顶堆保留前 K 名”：

```java
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int num : nums) {
        freq.put(num, freq.getOrDefault(num, 0) + 1);
    }

    PriorityQueue<int[]> heap =
            new PriorityQueue<>((a, b) -> Integer.compare(a[1], b[1]));

    for (Map.Entry<Integer, Integer> entry : freq.entrySet()) {
        heap.offer(new int[] {entry.getKey(), entry.getValue()});
        if (heap.size() > k) {
            heap.poll();
        }
    }

    int[] result = new int[heap.size()];
    for (int i = result.length - 1; i >= 0; i--) {
        result[i] = heap.poll()[0];
    }
    return result;
}
```

这段模板背后的重点是：  
**如果你只需要维护前 K 个，没必要把所有元素完整排序。**

这个思想和很多后端场景非常像，比如热点榜单、优先级任务、限量结果集等。

### 五、二分查找：不是只会“找数字”，而是会“找边界”

很多人对二分查找的理解停留在“在有序数组里找一个值”，这太窄了。面试里更常考的是边界二分，也就是：

- 第一个大于等于目标的位置
- 最后一个小于等于目标的位置
- 最小可行值
- 最大可行值

所以二分真正的关键词不是“某个数是否存在”，而是“条件是否具有单调性”。只要答案空间有单调性，二分就有机会成立。

先把最基础的边界模板背熟就够了：

```java
public int lowerBound(int[] nums, int target) {
    int left = 0;
    int right = nums.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;
}
```

你要真正掌握的不是这一段代码，而是这几个判断：

- 当前区间是左闭右开还是左闭右闭
- `mid` 命中后，答案在左边还是右边
- 你要找的是“存在性”还是“边界”

只要这三个问题想清楚，二分就不容易写乱。

### 六、树的 DFS / BFS：处理层次结构与路径问题

树题在后端面试里价值很高，因为它和真实业务的关系非常近。菜单树、组织架构、评论树、目录树、流程树，本质都是树结构。你如果只把树题当成算法练习，会低估它的重要性。

树题通常要先判断是用 DFS 还是 BFS：

- DFS 适合处理路径、递归结构、子树信息汇总
- BFS 适合处理层序遍历、最短层数、逐层扩展

如果题目一眼看上去是“按层输出”“一层一层看”，那通常 BFS 更自然。下面是最经典的层序遍历模板：

```java
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) {
        return result;
    }

    Queue<TreeNode> queue = new ArrayDeque<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int size = queue.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null) {
                queue.offer(node.left);
            }
            if (node.right != null) {
                queue.offer(node.right);
            }
        }
        result.add(level);
    }
    return result;
}
```

这段代码体现的是一个非常稳定的结构：

- 队列里放当前层待处理节点
- 每轮先记住当前层大小
- 循环处理这一层，再把下一层节点放进去

以后你遇到“层数”“每层结果”“最短步数”这类题时，都可以先往 BFS 上靠。

### 七、单调栈、回溯、动态规划：先知道它们解决什么问题

这三类题通常比前面更难一些，所以第 1 课不展开到很细，但你至少要先知道它们各自解决什么矛盾。

单调栈最常见的信号是：

- 下一个更大元素
- 下一个更小元素
- 每个元素左边或右边最近的更大值 / 更小值

它适合解决“最近满足条件的位置”问题，本质是用栈维护一个有序关系。

回溯最常见的信号是：

- 全排列
- 子集
- 组合
- 括号生成

它的本质是枚举所有可能路径，并在递归回退时撤销选择。

动态规划最常见的信号是：

- 最多
- 最少
- 最优
- 第 i 个状态依赖前面若干状态

它的本质不是套模板，而是定义状态、写出转移。DP 是很多人的难点，所以后面值得单独展开。

### 八、面试里怎么讲，比分散写代码更重要

现在把前面的东西串起来。后端面试里的算法题，最稳的表达顺序通常是：

1. 先复述题目，确认输入输出和约束
2. 判断这题更像哪种套路
3. 先说一个直观但可能不够优的办法
4. 指出瓶颈，并说明为什么该换成某个数据结构
5. 再写核心 Java 代码
6. 最后走一个样例，检查边界和复杂度

这个顺序很重要，因为它能体现你的判断过程。面试官真正信任的，不是“你恰好写对了”，而是“你知道为什么这样做”。对后端工程师来说，这比刷出一个华丽但说不清楚的答案更重要。

你也会发现，这种学习方式和你想要的方向是一致的：  
先有文字总结和套路图谱，再用少量 Java 模板作为抓手。  
这比把文档做成代码仓库，更有利于你后面反复复习。

## 小结（3-5 条关键点）

1. 高频算法题并不是完全分散的，绝大多数都可以归到少数几个经典套路里。
2. 学算法更有效的方式，是先记“题型信号”和“适用条件”，再记 Java 模板。
3. 哈希、滑动窗口、前缀和、堆、二分、树遍历，是后端面试里最值得优先掌握的主干。
4. 代码在这类学习里是辅助，不是主体；真正决定你能不能迁移到新题上的，是套路理解。
5. 面试表达顺序应该是：识别题型、解释思路、说明复杂度、最后再落代码。
