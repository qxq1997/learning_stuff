# 虚拟内存：地址空间、分页、分段、页表、MMU 与 TLB

## 为什么需要虚拟内存

虚拟内存给每个进程一个独立、连续、私有的地址空间。它带来几个关键好处：

- 隔离：进程不能随便访问其他进程内存。
- 保护：页表项可以标记读、写、执行、用户/内核权限。
- 扩展：进程看到的虚拟地址空间可以大于物理内存。
- 简化：程序不必关心自己被装载到哪一块物理内存。
- 共享：不同进程可以把不同虚拟地址映射到同一物理页，实现共享库、共享内存和写时复制。

没有虚拟内存，应用直接使用物理地址，多进程隔离和动态装载都会非常困难。

## 虚拟地址如何翻译成物理地址

在分页系统中，虚拟地址通常被拆成两部分：

```text
虚拟地址 = 虚拟页号 + 页内偏移
```

翻译过程：

1. CPU 产生虚拟地址。
2. MMU 取出虚拟页号。
3. MMU 查 TLB，命中则直接拿到物理页框号。
4. TLB 未命中时，按页表查找虚拟页号对应的物理页框号。
5. 物理页框号加页内偏移，得到物理地址。
6. 如果页表项无效或权限不满足，触发异常进入内核。

页内偏移不需要翻译，因为虚拟页和物理页大小相同，页内位置保持不变。

## 页表项里通常有什么

页表项不只是物理页号，还包含很多控制位：

- Present：页面是否在内存。
- Read/Write：是否可写。
- User/Supervisor：用户态是否可访问。
- Execute disable：是否禁止执行。
- Accessed：是否被访问过，可辅助页面置换。
- Dirty：是否被写过，可决定回收前是否要写回。
- Copy-on-write 标记：写入时触发复制。

这些控制位让内核能用同一套页表机制实现保护、缺页、共享、懒分配和写时复制。

## TLB 为什么重要

页表在内存里。如果每次访存都先访问页表，再访问真实数据，成本会非常高。TLB 是地址翻译缓存，保存近期虚拟页到物理页的映射。

TLB 命中：

- 地址翻译很快。

TLB 未命中：

- 需要走页表查询，甚至多级页表查询。

进程切换可能影响 TLB，因为不同进程的虚拟页号可能相同但映射不同。如果 CPU 支持地址空间标识，可以减少全量刷新，否则切进程会让 TLB 命中率下降。

## 多级页表为什么存在

如果用单级页表覆盖整个虚拟地址空间，很多进程根本用不到的大量地址也要占页表空间。多级页表把页表分层，未使用的虚拟地址范围可以不分配下级页表。

代价是：

- 地址翻译可能需要多次内存访问。
- 更依赖 TLB 缓存。

所以多级页表省内存，TLB 保性能，两者通常一起出现。

## 分页和分段的区别

| 机制 | 切分方式 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 分页 | 固定大小页 | 无外部碎片，便于换入换出 | 可能有内部碎片，页表管理复杂 |
| 分段 | 按代码、数据、栈等逻辑段 | 符合程序逻辑，权限控制直观 | 容易产生外部碎片 |
| 段页式 | 先分段，再分页 | 兼顾逻辑和物理管理 | 结构更复杂 |

分页更适合现代通用内存管理，因为固定大小的页让分配、回收、置换都更简单。

## 进程地址空间布局

典型用户态地址空间从低到高可理解为：

```text
低地址
保留区
代码段
数据段
BSS 段
堆
文件映射区
栈
内核映射区域
高地址
```

各段含义：

- 代码段：可执行指令，通常只读可执行。
- 数据段：已初始化全局变量和静态变量。
- BSS 段：未初始化全局变量和静态变量，装载时清零。
- 堆：动态分配内存，通常向高地址增长。
- 文件映射区：动态库、共享内存、`mmap` 映射。
- 栈：函数调用、局部变量、返回地址，通常向低地址增长。

保留低地址区域可以让空指针等低地址访问更容易被发现。

## 缺页异常在虚拟内存中的位置

缺页异常不是单纯的错误，而是虚拟内存懒加载的关键机制。常见场景：

- `malloc` 申请了虚拟地址，但第一次写入时才分配物理页。
- 文件映射页第一次访问时才从磁盘加载。
- `fork` 后父子进程共享只读页，写入时触发写时复制。
- 访问非法地址或越权访问时，内核无法修复，进程收到异常信号。

## 面试常见追问

**虚拟内存比物理内存大，程序为什么还能运行？**

因为不是所有虚拟页都需要同时在物理内存里。只有活跃工作集需要常驻内存，不活跃页面可以尚未分配、映射文件、或被换出。

**分页为什么没有外部碎片？**

物理内存按固定大小页框分配，任意虚拟页都能映射到任意空闲物理页框，不要求连续大块物理空间。

**为什么会有内部碎片？**

页是最小分配和映射单位。如果只用几百字节，也可能占一个 4KB 页的一部分，页内未用空间就是内部碎片。
