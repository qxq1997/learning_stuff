# 美团结算中心系统设计学习路径

## 学习目标

这条学习路径不是为了背几个“结算”“清算”“对账”的定义，而是为了把你带到这样一种水平：

- 你能把外卖结算系统讲成一个完整的资金系统，而不是一个“算佣金的模块”
- 你能分清规则计算、账务记账、清算、付款、对账各自的边界
- 你能从工程视角解释为什么这类系统最怕乱序、重复、退款、跨周期冲销和规则变更
- 你能在面试里像真正做过交易系统的人一样，把系统拆清楚、把风险点说到位

- 补充可观测性和业务监控，这在资金系统里非常关键

另外要强调一句：  
下面的内容是**基于通用互联网外卖/本地生活清结算系统的架构抽象**，用于学习和系统设计训练，不代表美团内部真实实现细节，也不应假设自己知道任何公司内部机密。

## 知识图谱

这个主题建议按 4 个层次理解：

1. 业务层
   - 一单钱到底怎么分
   - 参与方有哪些
   - 用户、商家、骑手、平台、渠道之间的资金关系是什么
   - 为什么“订单完成后直接打钱”会出事故

2. 核心系统层
   - 订单事件接入
   - 规则引擎
   - 账务引擎
   - 清算与结算单
   - 付款与对账

3. 分布式工程层
   - 幂等
   - 乱序
   - 状态机
   - 逆向交易
   - 补偿
   - 最终一致

4. 架构与面试表达层
   - 模块怎么拆
   - 技术怎么选
   - 指标怎么监控
   - 面试里怎么讲得“像做过”

## 课程安排

1. `01_外卖结算中心全景与核心链路.md`
   - 先把“它到底在解决什么问题”讲透
   - 重点建立全局视角，防止一上来陷进数据库表或 MQ 细节

2. `02_规则引擎、账务分录与账户模型.md`
   - 这是系统的心脏
   - 重点讲钱怎么算、怎么算完之后怎么记账、为什么不能直接改历史金额

3. `03_消息乱序、幂等、一致性与补偿.md`
   - 这是结算系统最有分布式味道的一课
   - 重点讲为什么“消息大概率不干净”，以及系统如何扛住脏现实

4. `04_技术选型、可观测性与面试回答框架.md`
   - 把前面内容工程化
   - 重点讲组件选择、监控体系和系统设计题回答套路

5. `04b_结算中心如何应对大流量.md`
   - 这是第 4 课的补充课
   - 重点讲峰值流量下的削峰、异步化、热点隔离、批处理、限流降级和账务保护

## 推荐学习方式

- 先完整读第 1 课，不要急着看技术细节
- 第 2 课要重点理解“规则版本 + 订单快照 + 分录追加”这组组合
- 第 3 课建议反复看，因为很多面试高低差就在这里
- 第 4 课用来把前面内容组织成“可表达的答案”

## 学完后你应该能回答的问题

- 为什么外卖结算系统不是“订单表 + 几个金额字段”
- 为什么这类系统一定要保留订单快照和规则版本
- 为什么账务系统偏好 append-only 分录，而不是 update 历史账
- 为什么消息乱序不能只靠 MQ 保证顺序解决
- 为什么资金系统通常偏向本地事务 + Outbox + 补偿 + 对账，而不是全链路强事务
- 为什么“业务监控”在结算系统里比很多普通系统更重要
