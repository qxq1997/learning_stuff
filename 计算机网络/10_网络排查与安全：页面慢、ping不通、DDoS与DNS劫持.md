# 第 10 课：网络排查与安全：页面慢、ping 不通、DDoS 与 DNS 劫持

## 学习目标

- 把“页面加载慢”拆成 DNS、TCP、TLS、HTTP、服务端处理、传输多个阶段。
- 解释 ping 不通但 HTTP 能访问的原因。
- 理解 DDoS、SYN Flood、DNS 劫持的基本原理和防护方向。
- 建立网络问题排查的分层思路。

## 页面加载慢怎么排查

不要一上来就说“可能是网络问题”。应该按链路分段：

1. DNS 是否慢
   - 域名解析耗时高。
   - DNS 缓存失效或递归解析慢。
   - DNS 返回了错误地域的 IP。
2. TCP 建连是否慢
   - SYN 重传。
   - 服务端半连接或全连接队列满。
   - 中间网络丢包。
3. TLS 是否慢
   - 证书链过长。
   - TLS 版本和密码套件协商慢。
   - 没有会话复用。
4. HTTP 请求是否慢
   - 请求头或 body 过大。
   - 代理排队。
   - 上游超时。
5. 服务端处理是否慢
   - 应用线程池满。
   - 数据库慢查询。
   - 缓存未命中。
6. 响应传输是否慢
   - 响应体过大。
   - 带宽不足。
   - 丢包重传。
   - 客户端接收慢。

可以用浏览器 Network Timing、服务端 access log、Nginx upstream time、tcpdump、curl、dig、traceroute、ss 等工具逐段定位。

## ping 不通但 HTTP 能访问

ping 使用 ICMP，HTTP 使用 TCP。ping 不通不代表 TCP 一定不通。

可能原因：

- 目标主机或防火墙禁用了 ICMP。
- 云厂商安全组禁止 ICMP，但允许 80/443。
- 中间网络丢弃 ICMP。
- 域名解析到不同 IP，ping 和 HTTP 访问目标不一致。
- HTTP 经过代理、CDN、负载均衡，而 ping 的是另一个节点。

所以排查时应直接测试业务端口：

```text
curl -v https://example.com
nc -vz example.com 443
telnet example.com 443
```

ping 只能说明 ICMP 探测结果，不能替代业务协议探测。

## DDoS

DDoS 是分布式拒绝服务攻击，攻击者利用大量机器向目标发起流量或请求，让目标资源耗尽。

常见类型：

- 流量型攻击：占满带宽。
- 协议型攻击：利用 TCP、UDP、ICMP 等协议消耗连接和协议栈资源。
- 应用层攻击：大量 HTTP 请求打满应用、数据库、缓存。

防护思路：

- CDN 和高防清洗。
- 限流、黑白名单、验证码或挑战。
- Anycast 和流量调度。
- 连接数、请求速率、异常 User-Agent 和路径监控。
- 缓存静态内容，减少源站暴露。

## SYN Flood

SYN Flood 是典型协议层攻击。

攻击者发送大量 SYN，但不完成第三次握手，服务端半连接队列被占满，正常用户连接无法进入。

缓解方式：

- 开启 SYN Cookie。
- 调整半连接队列和重试参数。
- 前置负载均衡或高防。
- 限制单源连接速率。
- 监控 SYN_RECV 状态数量。

注意：SYN Cookie 是内核层缓解手段，不是全量 DDoS 防护。大流量攻击仍需要上游清洗。

## DNS 劫持

DNS 劫持是攻击者篡改域名解析结果，把用户导向错误 IP。

表现可能是：

- 打开正确域名却跳到广告页。
- HTTPS 证书错误。
- 同一域名在不同网络环境解析结果异常。
- 某些地区访问异常。

排查方向：

- 对比不同 DNS 解析结果。
- 检查权威 DNS 配置。
- 检查本地 hosts、路由器 DNS、运营商 DNS。
- 检查证书域名和证书链。
- 使用 DoH、DoT 或可信递归 DNS 做对照。

防护方向：

- HTTPS 证书校验。
- HSTS。
- 可信 DNS。
- 关键域名解析监控。
- 客户端对关键服务做证书或公钥约束时要谨慎设计更新机制。

## 网络排查的分层口诀

可以按这条链路排：

```text
域名是否解析对
IP 是否能路由到
端口是否能连上
TLS 是否能握手
HTTP 是否有响应
上游是否超时
应用是否处理慢
数据库和缓存是否拖后腿
```

常用观测点：

- DNS：`dig`、`nslookup`。
- 路由：`traceroute`、`mtr`。
- 端口：`nc`、`telnet`、`ss`。
- HTTP：`curl -v`、浏览器 Network。
- 抓包：`tcpdump`、Wireshark。
- 服务端：access log、error log、upstream time、应用日志。

## 小结

- 页面慢要分段排查，不要笼统归因网络。
- ping 不通只说明 ICMP 不通，业务 TCP 端口可能仍然可用。
- DDoS 分流量型、协议型、应用层，防护要结合清洗、限流、缓存和监控。
- SYN Flood 主要打半连接队列，SYN Cookie 可以缓解但不是万能。
- DNS 劫持通过篡改解析结果误导用户，HTTPS 证书校验是重要保护层。
