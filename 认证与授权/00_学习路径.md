# 认证与授权 学习路径与知识图谱

## 学习定位

`JWT`、`OAuth`、`SSO`、`RBAC` 这些词，很多后端工程师都听过，也经常在项目、面试、网关、登录系统里遇到。  
但真正容易混乱的地方在于：

- 有的概念在解决“你是谁”
- 有的概念在解决“你能做什么”
- 有的概念在解决“登录态怎么保存”
- 有的概念在解决“多个系统怎么共享身份”
- 有的概念在解决“第三方应用如何安全拿到授权”

所以学这一块，最怕的不是完全不会，而是脑子里有很多熟词，但边界混在一起。

这条学习路径的目标，是把认证与授权这套知识拆成几层稳定的心智模型，让你最后不只是会背定义，而是能在工程和面试中真正讲清楚：

- `Authentication`、`Authorization`、`AuthN`、`AuthZ`、鉴权、权限控制到底分别指什么
- `Cookie`、`Session`、`Token`、`JWT` 分别适合什么场景
- `OAuth2` 和 `OIDC` 的关系是什么，为什么“第三方登录”常常会提到它们
- `SSO` 是产品能力还是协议能力，企业统一身份到底怎么做
- `RBAC`、`ABAC`、`ACL`、数据权限这些权限模型为什么会让系统越来越复杂

## 这套知识到底在解决什么

你可以先把整张图谱压缩成四个问题：

1. 你是谁
   - 用户名密码
   - 手机验证码
   - 多因子认证
   - 第三方登录
2. 你登录之后，系统怎么记住你
   - Cookie
   - Session
   - Token
   - JWT
3. 你能访问什么
   - 角色权限
   - 资源权限
   - 数据权限
   - 租户隔离
4. 多系统、多应用、多组织之间怎么共享身份和授权
   - OAuth2
   - OIDC
   - SSO
   - 联邦身份

## 知识图谱（从基础到进阶）

1. 先分清认证、授权、鉴权、权限控制
   - 认证是确认身份
   - 授权是授予能力
   - 鉴权是请求到来时做校验
   - 权限控制是整套长期治理体系
2. 登录态保存机制
   - Cookie
   - Session
   - Token
   - JWT
   - Refresh Token
3. 开放授权与统一身份
   - OAuth2
   - OIDC
   - 第三方登录
   - 授权码模式
4. 企业统一登录与多系统打通
   - SSO
   - IdP
   - SP
   - 登录态传播
   - 单点登出
5. 权限模型与工程落地
   - RBAC
   - ABAC
   - ACL
   - 数据权限
   - 菜单、按钮、API、行列级权限
6. 常见补充认证方案
   - Basic Auth
   - API Key
   - HMAC 签名
   - mTLS
   - 设备身份与服务身份
7. 安全风险与面试高频坑
   - Token 泄漏
   - CSRF
   - XSS
   - Session 固定攻击
   - JWT 无法主动失效
   - OAuth 回调劫持与状态参数

## 为什么这样排序

认证授权最容易学乱的原因，是很多人一上来就直接看 `OAuth2` 流程图，或者一上来就被 `JWT` 的 header、payload、signature 三段结构吸走注意力。

但如果你没有先搞清楚：

- 到底什么是认证，什么是授权
- 到底谁在颁发身份，谁在校验权限
- 到底 Cookie / Session / Token 是“状态载体”还是“协议本身”

那后面所有流程图都会越看越像。

所以顺序必须是：

1. 先建立概念边界
2. 再理解登录态保存
3. 再看开放授权和统一身份
4. 最后再进入企业级权限模型和复杂落地

## 学完后你应该能回答的问题

- 为什么说“JWT 不是登录协议，而是一种令牌格式”。
- 为什么 `OAuth2` 解决的是授权委托，而不是最初意义上的“登录”。
- 为什么很多公司做 SSO 时，背后还要再配一个 `OIDC` 或 `SAML`。
- 为什么 `RBAC` 一开始很好用，业务复杂后又会开始觉得不够。
- 为什么菜单权限、按钮权限、接口权限、数据权限不能简单画等号。
- 为什么认证中心、用户中心、权限中心、网关、业务服务之间要有清晰边界。

## 面试表达建议

如果面试官问“你讲一下 JWT、OAuth、SSO”，不要按词典式一个个背定义，而是建议按下面的主线回答：

1. 先把认证和授权分开
2. 再讲登录态的几种保存方式：Session 和 JWT
3. 再讲第三方授权和统一身份：OAuth2、OIDC、SSO
4. 最后补权限模型：RBAC、ABAC、数据权限

这样回答的好处是，你讲的是一套系统，而不是四个孤立名词。
