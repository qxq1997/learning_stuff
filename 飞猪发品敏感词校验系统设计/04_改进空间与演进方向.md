# 第 4 课：改进空间与演进方向

面试里"如果让你重做，你会怎么改进"这个问题最能体现深度，因为它考的是你对系统局限的**反思能力**。一个只会讲"我做了什么"的候选人和一个能讲"我做的方案天花板在哪、下一步往哪走"的候选人，水平差距一目了然。

这一课五个方向，第一个（规则 + LLM）和你的 RAG / Agent 方向最契合，建议作为重点，能展开讲到落地细节。

## 改进 1：引入语义级检测（规则 + LLM 混合管线）

### 现有方案的天花板

现有方案本质是**字符串匹配 + 规则系统**。它的优势是高 precision、低延迟、可解释（命中哪个词清清楚楚），但它有一个根本局限：**处理不了"语义违规"**。

比如这几句：

- "我们的产品没人能比"
- "保证你玩得开心到飞起"
- "跟着我们走绝对不会后悔"

它们**没有任何一个明确的敏感词**，但语义上违反广告法（变相的绝对化、夸大承诺）。规则系统对这类文本束手无策——你不可能把所有"语义上违规但用词正常"的句子都枚举成词。

### 解法：规则 + LLM 双轨

引入**小参数量 LLM**（或 embedding + 分类头微调）做二级语义判别，和规则系统组成双轨：

- **规则轨**：命中直接拦截。高 precision、低延迟，扛住主流量。
- **语义轨**：规则**没**命中的文本，走一次低成本的 LLM 语义扫描，判断有没有"语义违规"。

落地的关键设计：

1. **分流而非全量**：不是每段文本都过 LLM（成本和延迟扛不住），而是只对规则未命中的文本做语义判别，且在**高风险类目**（医美、保健、金融跨界）强制启用，低风险类目可以采样或关闭。
2. **异步而非同步**：LLM 判别走异步，不阻塞发品。判风险高的，回写给商家"建议复核"，而不是硬拦（语义判别的 precision 不如规则，硬拦误伤大）。
3. **沉淀样本**：LLM 判为高风险的文本，沉淀为新的训练样本，反哺模型迭代。

这条线的简历价值：**把传统规则系统升级为"规则 + 模型"混合的内容安全管线**——规则保证延迟和可解释，模型补语义盲区，两者各司其职。这正是当下内容安全系统演进的主流方向。

## 改进 2：主动学习闭环

系统每天都在产生高价值的标注信号，现在大多浪费了：

- 商家每次**申诉**（"这个不该拦"）→ 负样本
- 运营每次**加白名单** → 负样本
- 每次命中后商家**修改文本的 diff**（他把违规的部分改成了什么）→ 正负样本对

把这些信号收集起来，定期训练 / 微调判别模型，形成闭环：

```
线上判别 → 人工反馈（申诉/加白/修改） → 收集为样本 → 模型更新 → 线上判别
```

这让系统**越用越准**，而不是靠运营手工堆规则。和改进 1 是天然配套——LLM 判别需要持续的高质量样本，主动学习闭环正好供给。

## 改进 3：变体识别系统化

第 3 课的归一化（去标点、繁简、全半角）只是最基础的变体对抗。更系统的做法：

- **谐音**：用拼音库归一化，"fa"和"发"映射到同一拼音再匹配
- **拆字 / 形近字**：用字形相似度库，把形近字归一
- **模糊匹配**：用字符级 embedding 做向量相似度匹配，能抓住归一化规则覆盖不到的新变体

变体对抗是猫鼠游戏，规则永远滞后于对抗者。embedding 模糊匹配的价值是从"枚举变体"升级为"理解相似"。

## 改进 4：词库治理自动化

现在哪些词有效、哪些该下线，靠运营手工看报表。可以做一个**评分系统**：综合这几个维度自动给词打分、给出状态建议——

- 命中率（高 = 有效）
- 申诉率（高 = 可能误伤）
- 加白率（高 = 规则有问题）
- 误判率

根据评分自动建议词的状态：**保留 / 下线 / 调整等级**。把第 1 课讲的"命中统计闭环"从"给人看的报表"升级为"能自动给建议的治理系统"。

## 改进 5：多语言支持

飞猪有**出境游**业务，商家可能写英文、日文、繁体描述。需要：

- 多语言词库
- 多语言分词策略：中文 AC 自动机不需要分词（按字符匹配），但**英文需要词边界判断**（否则 "class" 会误命中 "ass"），**日文要处理假名变体**（平假名/片假名/汉字混写）

这是把单语言系统扩展到国际化场景的必经之路。

## 怎么在面试里讲这一课

不要五个点平均用力地念一遍。推荐：

- **重点讲改进 1**（规则 + LLM），把"规则未命中才走语义轨""异步不硬拦""高风险类目强制启用""沉淀样本"这几个落地细节讲透——这是体现你既懂传统系统、又懂 LLM 落地的最佳素材。
- 改进 2（主动学习闭环）作为改进 1 的配套顺带讲，体现你想的是**系统而不是单点**。
- 改进 3/4/5 各一句话带过，体现你还有储备，但不展开。

## 检查站

1. 规则系统处理不了哪一类违规？举一个"没有敏感词但语义违规"的例子。
2. 规则 + LLM 双轨里，为什么 LLM 只对"规则未命中"的文本跑、且要异步、且不硬拦？
3. 系统里有哪些天然的标注信号可以用来做主动学习？它们分别是正样本还是负样本？
4. 词库治理评分系统会用哪几个指标？某个词"加白率很高"该建议什么操作？
5. 为什么中文 AC 自动机不用分词，而英文需要词边界判断？不做会出什么问题？

答完进入第 5 课，讲真做这个系统时线上会爆的故障形态。
