## 重点关注的理解难点

- 容易把"敏感词校验"理解成一个简单的关键词过滤器，而不是可运营、可观测的内容安全系统
- 容易讲不清"为什么不直接用集团第三方"，导致自建的必要性立不住
- 容易只会说"用 AC 自动机"，但讲不清它相比朴素匹配的复杂度优势、以及节点挂元数据的设计
- 容易忽视词库热更新的"全量重建 + volatile 原子切换"以及它的 CPU 抖动代价
- 容易忘记白名单优先级和变体归一化这两个"准确性"关键设计
- 容易把第三方调用当成普通依赖，忽视超时 / 熔断 / 降级
- 容易只讲架构，答不出内存膨胀、前后端词库不一致、对抗绕过这些生产问题
- 改进方向里，容易讲不出"规则 + LLM 混合管线"的分流 / 异步 / 不硬拦 / 沉淀样本这些落地细节

## 评估标准（用于自适应推进）

- 完全掌握：答案准确，能举一反三 -> 进入下一课

- 部分理解：概念混淆或有明显误区 -> 生成补充课（如 `03b_补充.md`）
- 尚未理解：关键问题回答错误或不知道 -> 回到更基础层重建理解
